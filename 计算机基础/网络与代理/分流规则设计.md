# 分流规则设计

> 让流量"各走各的路"——该直连的直连，该代理的代理

## 一、分流的核心思想

```
分流 = 流量分类 + 路由决策

流量分类依据：
├─ 域名（DOMAIN, DOMAIN-SUFFIX, DOMAIN-KEYWORD）
├─ IP（IP-CIDR, GEOIP）
├─ 进程（PROCESS-NAME）
├─ 端口（DST-PORT, SRC-PORT）
└─ 规则集（RULE-SET）
```

## 二、Clash 规则类型详解

### 基础规则类型

| 规则类型 | 匹配方式 | 示例 | 说明 |
|----------|----------|------|------|
| `DOMAIN` | 精确域名 | `DOMAIN,www.baidu.com,DIRECT` | 只匹配 www.baidu.com |
| `DOMAIN-SUFFIX` | 域名后缀 | `DOMAIN-SUFFIX,baidu.com,DIRECT` | 匹配 *.baidu.com |
| `DOMAIN-KEYWORD` | 域名关键词 | `DOMAIN-KEYWORD,google,PROXY` | 匹配包含 google 的域名 |
| `IP-CIDR` | IP 段 | `IP-CIDR,192.168.0.0/16,DIRECT` | 匹配内网 IP |
| `GEOIP` | IP 地理位置 | `GEOIP,CN,DIRECT` | 匹配中国 IP |
| `GEOSITE` | 域名地理位置 | `GEOSITE,cn,DIRECT` | 匹配中国网站 |
| `PROCESS-NAME` | 进程名 | `PROCESS-NAME,chrome.exe,PROXY` | 匹配特定程序 |
| `RULE-SET` | 规则集 | `RULE-SET,cn-direct,DIRECT` | 使用外部规则集 |
| `MATCH` | 兜底规则 | `MATCH,PROXY` | 匹配所有剩余流量 |

### 规则优先级

```
规则从上到下匹配，第一条匹配的规则生效

rules:
  - DOMAIN-SUFFIX,google.com,PROXY      # 1. 先匹配这个
  - DOMAIN-SUFFIX,google.cn,DIRECT      # 2. 再匹配这个
  - GEOIP,CN,DIRECT                     # 3. 中国 IP 直连
  - MATCH,PROXY                         # 4. 兜底：其他全走代理
```

**关键点：** 更具体的规则要放在更前面！

## 三、推荐的三种分流策略

### 策略一：黑名单模式（默认直连）

适合：主要访问国内网站，偶尔需要代理

```yaml
rules:
  # 代理的网站
  - DOMAIN-SUFFIX,google.com,PROXY
  - DOMAIN-SUFFIX,github.com,PROXY
  - DOMAIN-SUFFIX,youtube.com,PROXY
  # ... 更多需要代理的域名

  # 其他全部直连
  - GEOIP,CN,DIRECT
  - MATCH,DIRECT
```

**优点：** 简单，国内访问快
**缺点：** 需要手动添加被墙网站

### 策略二：白名单模式（默认代理）

适合：主要访问国外网站

```yaml
rules:
  # 直连的网站
  - DOMAIN-SUFFIX,baidu.com,DIRECT
  - DOMAIN-SUFFIX,qq.com,DIRECT
  - DOMAIN-SUFFIX,taobao.com,DIRECT
  # ... 更多需要直连的域名

  # 国内 IP 直连
  - GEOIP,CN,DIRECT

  # 其他全部代理
  - MATCH,PROXY
```

**优点：** 大部分网站都能访问
**缺点：** 需要手动维护国内网站列表

### 策略三：规则集模式（推荐）

使用社区维护的高质量规则集，兼顾两者：

```yaml
rule-providers:
  # 直连域名
  direct-domain:
    type: http
    behavior: domain
    url: "https://cdn.jsdelivr.net/gh/Loyalsoldier/clash-rules@release/direct.txt"
    path: ./ruleset/direct.yaml
    interval: 86400

  # 直连 IP
  direct-ip:
    type: http
    behavior: ipcidr
    url: "https://cdn.jsdelivr.net/gh/Loyalsoldier/clash-rules@release/cncidr.txt"
    path: ./ruleset/cncidr.yaml
    interval: 86400

  # 代理域名
  proxy-domain:
    type: http
    behavior: domain
    url: "https://cdn.jsdelivr.net/gh/Loyalsoldier/clash-rules@release/proxy.txt"
    path: ./ruleset/proxy.yaml
    interval: 86400

rules:
  # 代理规则
  - RULE-SET,proxy-domain,PROXY

  # 直连规则
  - RULE-SET,direct-domain,DIRECT
  - RULE-SET,direct-ip,DIRECT

  # 国内 IP 直连
  - GEOIP,CN,DIRECT

  # 兜底
  - MATCH,PROXY
```

**优点：** 规则全面，自动更新
**缺点：** 依赖外部规则源

| 规则集 | 内容 | 策略 |

| `direct.txt` | 需要直连的域名 | DIRECT |

## 五、你的问题：为什么规则"有时有效有时无效"

### 原因分析

```
问题：你添加了 baidu.com 的直连规则，但访问仍然很慢

可能的原因：
│
├─ 1. DNS 问题（最常见）
│   └─ DNS 返回了错误的 IP（海外 CDN）
│   └─ 规则匹配的是域名，但实际连接的是 IP
│
├─ 2. 规则顺序问题
│   └─ 在你的规则之前，有其他规则匹配了
│   └─ 比如 DOMAIN-KEYWORD,*,PROXY
│
├─ 3. 子域名问题
│   └─ 你添加了 baidu.com
│   └─ 但实际访问的是 www.baidu.com 或其他子域名
│
├─ 4. 多域名问题
│   └─ 网页加载时会访问多个域名（CDN、统计、广告等）
│   └─ 其中一个走了代理，拖慢整体速度
│
└─ 5. 网站自身问题
    └─ 网站识别到你的代理 IP，故意降速
    └─ 这是你无法控制的
```

### 调试方法

在 Clash Verge 中查看连接日志：

1. 打开 Clash Verge
2. 点击「日志」或「连接」标签
3. 访问问题网站
4. 查看连接走的是哪个规则、哪个策略组

## 六、进阶：进程分流

如果你想让特定程序走特定策略：

```yaml
rules:
  # 浏览器走代理
  - PROCESS-NAME,chrome.exe,PROXY
  - PROCESS-NAME,msedge.exe,PROXY

  # 游戏直连
  - PROCESS-NAME,steam.exe,DIRECT
  - PROCESS-NAME,LeagueofLegends.exe,DIRECT

  # 系统更新直连
  - PROCESS-NAME,wuauserv.exe,DIRECT

  # 其他规则...
```

## 🎯 核心要点回顾

1. **规则从上到下匹配，第一条生效** - 顺序很重要
2. **使用规则集** - 比手动维护更全面、更及时
3. **DNS 配置是规则生效的前提** - 先解决 DNS 问题
4. **用日志功能调试** - 看清楚流量实际走的是哪条规则
