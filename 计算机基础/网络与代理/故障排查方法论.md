# 故障排查方法论

> 当问题出现时，如何快速定位根因

## 一、问题定位四步法

```
┌─────────────────────────────────────────────────────────────┐
│                    问题定位四步法                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一步：复现问题                                            │
│  ├─ 确定问题是什么（访问慢？无法访问？部分功能异常？）         │
│  ├─ 确定问题出现的条件（哪些网站？什么时间？）                 │
│  └─ 尝试找出规律                                             │
│                                                             │
│  第二步：隔离变量                                            │
│  ├─ 关闭 TUN 模式，问题是否消失？                            │
│  ├─ 关闭代理，问题是否消失？                                 │
│  ├─ 换一个节点，问题是否消失？                               │
│  └─ 换一个网络（如手机热点），问题是否消失？                   │
│                                                             │
│  第三步：分层诊断                                            │
│  ├─ DNS 层：DNS 解析是否正确？                               │
│  ├─ 规则层：流量走的是哪条规则？                             │
│  ├─ 节点层：代理节点是否正常？                               │
│  └─ 网络层：直连是否正常？                                   │
│                                                             │
│  第四步：修复验证                                            │
│  ├─ 应用修复方案                                            │
│  ├─ 验证问题是否解决                                        │
│  └─ 记录问题和解决方案                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、常用诊断工具

### 1. DNS 诊断

```bash
# 查询域名解析
nslookup baidu.com

# 指定 DNS 服务器查询
nslookup baidu.com 223.5.5.5
nslookup baidu.com 8.8.8.8

# 查看当前 DNS 服务器
ipconfig /all | findstr DNS

# 清除 DNS 缓存
ipconfig /flushdns

# 使用 Clash 的 DNS 测试
nslookup baidu.com 127.0.0.1
```

### 2. 网络连通性测试

```bash
# Ping 测试
ping baidu.com

# 追踪路由
tracert baidu.com

# 端口连通性测试
telnet baidu.com 443

# 使用 curl 测试
curl -I https://www.baidu.com
curl -I --proxy http://127.0.0.1:7890 https://www.baidu.com
```

### 3. Clash 诊断

在 Clash Verge 中：

| 功能 | 位置 | 用途 |
|------|------|------|
| 连接列表 | 底部「连接」标签 | 查看实时连接和匹配规则 |
| 日志 | 底部「日志」标签 | 查看详细处理过程 |
| 延迟测试 | 代理组旁边 | 测试节点延迟 |
| 规则测试 | 需要开启日志 | 看流量走哪条规则 |

### 4. 浏览器诊断

Chrome 内置工具：

```
chrome://net-internals/#dns     - DNS 状态和缓存
chrome://net-internals/#sockets - Socket 连接状态
chrome://net-internals/#proxy   - 代理设置
```

### 5. 在线工具

| 工具 | 网址 | 用途 |
|------|------|------|
| DNS Leak Test | https://dnsleak.com/ | 检测 DNS 泄露 |
| IP 查询 | https://ip.sb/ | 查看当前 IP |
| 网速测试 | https://www.speedtest.net/ | 测试网速 |
| CDN 检测 | https://www.whatsmydns.net/ | 全球 DNS 解析对比 |

## 三、常见问题诊断流程

### 问题 A：访问国内网站很慢

```
诊断流程：
│
├─ 1. 确认问题
│   └─ 访问 https://ip.sb/ 确认当前 IP
│   └─ 如果显示国内 IP 但仍然慢 → 可能是网站问题
│   └─ 如果显示海外 IP → 分流问题
│
├─ 2. 检查分流规则
│   └─ Clash Verge → 连接 → 找到该网站的连接
│   └─ 查看匹配的规则和策略组
│   └─ 如果走的是 PROXY → 规则问题
│   └─ 如果走的是 DIRECT → DNS 或网站问题
│
├─ 3. 检查 DNS
│   └─ nslookup 该网站域名
│   └─ 对比国内 DNS 和代理 DNS 的解析结果
│   └─ 如果 IP 是海外 → DNS 配置问题
│
├─ 4. 检查节点
│   └─ 切换到其他节点测试
│   └─ 如果某个节点正常 → 节点问题
│
└─ 5. 检查网站自身
    └─ 用手机 4G 访问同一网站
    └─ 如果手机也慢 → 网站本身问题
```

### 问题 B：无法访问某个网站

```
诊断流程：
│
├─ 1. 确认是代理问题还是网站问题
│   └─ 关闭代理，直接访问
│   └─ 能访问 → 代理配置问题
│   └─ 不能访问 → 网站问题或被墙
│
├─ 2. 检查是否被广告规则拦截
│   └─ Clash Verge → 连接 → 找到该连接
│   └─ 如果策略是 REJECT → 被广告规则拦截
│   └─ 解决：把该域名加入白名单
│
├─ 3. 检查 DNS
│   └─ nslookup 该域名
│   └─ 如果返回错误 IP → DNS 污染
│   └─ 解决：使用 DoH
│
└─ 4. 检查节点
    └─ 换一个节点测试
    └─ 如果某个节点能访问 → 节点 IP 被封
```

### 问题 C：某些应用无法联网

```
诊断流程：
│
├─ 1. 确认是否是 TUN 模式问题
│   └─ 关闭 TUN 模式，开启系统代理
│   └─ 如果恢复正常 → TUN 配置问题
│
├─ 2. 检查进程规则
│   └─ Clash Verge → 连接 → 按进程筛选
│   └─ 查看该进程的连接走什么规则
│
├─ 3. 检查是否是 fake-ip 问题
│   └─ 某些应用不支持 fake-ip
│   └─ 解决：切换到 redir-host 模式
│   └─ 或把相关域名加入 fake-ip-filter
│
└─ 4. 检查是否是 UDP 问题
    └─ 某些代理节点不支持 UDP
    └─ 解决：换一个支持 UDP 的节点
```

### 问题 D：游戏延迟高或掉线

```
诊断流程：
│
├─ 1. 确认游戏服务器位置
│   └─ 国内游戏 → 应该直连
│   └─ 国外游戏 → 可能需要代理
│
├─ 2. 检查分流规则
│   └─ 游戏域名是否正确分流
│   └─ 国内游戏应该 DIRECT
│
├─ 3. 检查 fake-ip
│   └─ 游戏通常需要真实 IP
│   └─ 解决：游戏域名加入 fake-ip-filter
│   └─ 或使用 redir-host 模式
│
└─ 4. 检查 UDP 支持
    └─ 游戏通常使用 UDP
    └─ 确认代理节点支持 UDP
```

## 四、日志分析技巧

### 开启 Debug 日志

```yaml
log-level: debug
```

### 常见日志模式

```
# 正常的 DNS 解析
[DNS] baidu.com --> 110.242.68.3

# 规则匹配
[RULE] DOMAIN-SUFFIX,baidu.com --> DIRECT

# 连接建立
[TCP] 127.0.0.1:12345 --> baidu.com:443 [DIRECT]

# 代理连接
[TCP] 127.0.0.1:12345 --> google.com:443 [ProxyGroup] [节点名]
```

### 问题日志模式

```
# DNS 解析失败
[DNS] baidu.com --> resolve error

# 规则未匹配（走到 MATCH）
[RULE] MATCH --> PROXY  # 如果是国内网站，说明规则不完善

# 连接超时
[TCP] connection timeout

# 节点不可用
[Proxy] 节点名: connect error
```

## 五、快速检查清单

当你遇到问题时，按这个清单快速检查：

```
□ 1. 当前 IP 是什么？（访问 ip.sb）
□ 2. DNS 解析是否正确？（nslookup）
□ 3. 连接走的是什么规则？（Clash 连接列表）
□ 4. 节点是否正常？（延迟测试）
□ 5. 关闭代理后是否正常？（排除代理问题）
□ 6. 换网络后是否正常？（排除网络环境问题）
□ 7. 规则是否最新？（更新规则集）
□ 8. DNS 缓存是否清除？（ipconfig /flushdns）
```

## 六、记录问题和解决方案

建议维护一个问题日志：

```markdown
## 问题记录

### 2026-03-02: 访问 bilibili.com 很慢

**现象：** 开启 TUN 模式后，bilibili 视频加载很慢

**诊断：**
1. 检查连接 → 走的是 DIRECT
2. 检查 DNS → 返回的是海外 CDN IP
3. 原因：DNS 走了代理，返回了不适合的 IP

**解决：**
- 在 nameserver-policy 中添加 bilibili.com 使用国内 DNS
- 或在 fake-ip-filter 中添加 *.bilibili.com

**验证：** 问题解决
```

## 🎯 核心要点回顾

1. **先隔离变量** - 确定是代理问题还是其他问题
2. **分层诊断** - DNS → 规则 → 节点 → 网络
3. **善用工具** - nslookup、Clash 日志、在线工具
4. **记录问题和解决方案** - 避免重复踩坑
5. **有些问题无法解决** - 网站主动限制代理 IP
