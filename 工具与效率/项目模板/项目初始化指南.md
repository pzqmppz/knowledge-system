## 1. 项目规划阶段

### 1.1 需求文档 (PRD)

在写代码之前，先完成 PRD（Product Requirements Document）：

```markdown
# 项目 PRD 模板

## 项目概述
- 项目名称
- 项目目标
- 目标用户

## 核心功能
1. 功能一：描述
2. 功能二：描述
3. 功能三：描述

## 非功能需求
- 性能要求
- 安全要求
- 可用性要求

## 技术约束
- 技术栈限制
- 第三方依赖
- 部署环境
```

### 1.2 技术方案设计

基于 PRD 设计技术方案：

```markdown
# 技术方案模板

## 架构设计
- 系统架构图
- 技术栈选型
- 模块划分

## 数据库设计
- ER 图
- 表结构设计
- 索引策略

## API 设计
- RESTful 规范
- 接口列表
- 认证方式

## 部署方案
- 环境划分
- CI/CD 流程
- 监控告警
```

## 2. 项目初始化阶段

### 2.1 创建项目结构

```bash
# 创建项目目录
mkdir my-project && cd my-project

# 初始化 Git
git init

# 创建标准目录结构
mkdir -p src tests docs scripts
mkdir -p .vscode .github/workflows

# 创建基础文件
touch README.md CLAUDE.md .env.example .gitignore
```

### 2.2 配置项目文件

复制模板文件并修改：

```bash
# 如果你有项目模板库
cp ~/templates/project/CLAUDE模板.md ./CLAUDE.md
cp ~/templates/project/ENV模板.md ./.env.example

# 或者从知识体系复制
cp [知识体系]/工具与效率/项目模板/CLAUDE模板.md ./CLAUDE.md
cp [知识体系]/工具与效率/项目模板/ENV模板.md ./.env.example
```

### 2.3 填写项目信息

编辑 `CLAUDE.md`，填写：
- 项目名称和类型
- 技术栈
- GitHub 仓库地址
- 需要的 Skills
- 敏感信息存放位置

编辑 `.env.example`，添加：
- 必需的环境变量
- 数据库连接配置
- 第三方服务密钥

### 2.4 创建实际环境变量

```bash
# 从模板创建实际配置
cp .env.example .env

# 编辑填入真实值（注意不要提交）
vim .env
```

### 2.5 配置 .gitignore

确保 `.gitignore` 包含：

```gitignore
# 环境变量
.env
.env.local
.env.*.local

# 密钥文件
*.key
*.pem
secrets/

# 依赖
node_modules/
vendor/

# 构建
dist/
build/
*.log

# IDE
.vscode/
.idea/
*.swp
```

## 3. 开发准备阶段

### 3.1 初始化开发环境

```bash
# Node.js 项目
npm init -y
npm install

# Python 项目
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Rust 项目
cargo init
```

### 3.2 配置开发工具

创建 `.vscode/settings.json`：

```json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "claudeCode.selectedModel": "glm-4.7"
}
```

创建 `.vscode/extensions.json`（推荐扩展）：

```json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "anthropic.claude-code"
  ]
}
```

### 3.3 设置 CI/CD

创建 `.github/workflows/ci.yml`：

```yaml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm install
      - run: npm test
      - run: npm run build
```

### 3.4 创建 README.md

```markdown
# 项目名称

> 项目简短描述

## 快速开始

\`\`\`bash
npm install
npm run dev
\`\`\`

## 功能特性

- 特性一
- 特性二
- 特性三

## 文档

- [架构设计](docs/ARCHITECTURE.md)
- [API 文档](docs/API.md)
- [部署指南](docs/DEPLOYMENT.md)

## 许可证

MIT
```

## 4. 文件清单

### 必需文件

| 文件 | 作用 | 是否提交 |
|------|------|----------|
| `CLAUDE.md` | Claude Code 配置 | ✅ |
| `.env.example` | 环境变量模板 | ✅ |
| `.env` | 实际环境变量 | ❌ |
| `.gitignore` | Git 忽略规则 | ✅ |
| `README.md` | 项目说明 | ✅ |

### 推荐文件

| 文件 | 作用 | 是否提交 |
|------|------|----------|
| `docs/PROJECT_INIT.md` | 初始化记录 | ✅ |
| `docs/ARCHITECTURE.md` | 架构设计 | ✅ |
| `docs/API.md` | API 文档 | ✅ |
| `docs/DEPLOYMENT.md` | 部署方案 | ✅ |
| `.vscode/settings.json` | 编辑器配置 | ✅ |
| `.github/workflows/` | CI/CD 配置 | ✅ |

### 密钥文件（不提交）

| 文件 | 存放位置 |
|------|----------|
| SSH 密钥 | `~/.ssh/project_key` |
| API 密钥 | `~/.config/project/api_keys.json` |
| 云凭证 | `~/.aws/credentials` |
