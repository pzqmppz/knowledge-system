### 核心知识/价值点

#### 技术维度

**关键认知**：

1. **代理式编程的本质**
   - Workflow（工作流）：人类定义步骤
   - Agent（代理）：模型决定如何使用工具
   - 模型在驾驶座上，人类在循环中审批

2. **Claude Code 的能力**
   - 读取、编辑、写入文件
   - 运行系统命令
   - 访问互联网
   - 使用 MCP 工具
   - Subagents 专业化分工

3. **MCP 协议的意义**
   - 标准化的工具连接协议
   - Claude Code 既是客户端也是服务器
   - 企业工具的统一接入层

#### 产品/商业维度

**主流程图谱**：

```
内部原型 → 团队使用 → 公司采用 → 对外发布
     |          |          |          |
   笔记自动化  日报到    生产力    4亿美元
     ↓          ↓         震撼      年收入
  给bash工具  → 开始编码   入职缩短
     ↓                    几周到几天
  意外发现
```

**核心洞察提炼**：

1. **产品跟随模型哲学**
   - 传统：构建完美产品
   - 新范式：构建最简接口让模型展现
   - 因为模型进化太快，任何产品都跟不上

2. **终端的优势**
   - 通用性：跨平台、跨 IDE
   - 低耦合：不依赖特定系统
   - 工程师偏好：尊重个人选择

3. **内部使用是最好的验证**
   - Anthropic 全员日报到
   - 技术入职从几周到几天
   - 产品能力来自真实使用场景

**隐含假设**：
- 模型能力会持续快速进化
- 代理式编程是未来范式
- 代码是 AI 与世界交互的最佳方式
