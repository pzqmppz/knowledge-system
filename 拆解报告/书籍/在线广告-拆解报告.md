## 第二轮：血肉解剖 | Deep Dissection

"凭什么这么说"

### 论证链

```
在线广告需求
    ↓
需要高效匹配机制
    ↓
RTB实时竞价架构 (DSP-SSP-DMP)
    ↓
核心算法: CTR预估 + 用户定向
    ↓
工程挑战: ≤100ms响应 + 高并发
    ↓
系统优化: 特征工程 + 模型迭代 + 反作弊
```

### 关键证据

1. **RTB时效性要求**
   - 用户等待容忍度 < 2秒
   - 竞价决策时间 ≤ 100ms
   - 这决定了系统架构必须高度优化

2. **CTR预估的商业价值**
   - CTR提升10% → eCPM提升10%
   - 直接影响广告收入
   - 百度/阿里等公司核心算法

3. **反作弊的必要性**
   - 虚假流量导致30%+损失
   - 点击欺诈普遍存在
   - 需要多维度检测机制

### 隐形假设

- **假设1**：用户行为数据可以准确预测点击率
- **假设2**：实时竞价机制能够实现市场均衡
- **假设3**：用户定向不会引发严重的隐私担忧
- **假设4**：广告主和媒体方的利益可以通过算法平衡

### 边界条件

| 何时失效 | 说明 |
|----------|------|
| 隐私法规收紧 | GDPR/CCPA限制数据使用 |
| 广告拦截普及 | 20%+用户使用广告拦截 |
| 品牌安全事件 | 广告展示在不当内容旁边 |
| 移动端限制 | IDFA/GAID变更影响定向 |

## 核心概念归档

### 概念 1：RTB (Real-Time Bidding)

- **定义**：实时竞价，程序化广告的核心机制
- **应用场景**：展示广告、视频广告、移动广告
- **关键指标**：竞价响应时间 ≤ 100ms
- **相关内容**：[[DSP]]、[[SSP]]、[[AdXchange]]

### 概念 2：CTR预估

- **定义**：点击率预估，预测用户点击广告的概率
- **技术栈**：逻辑回归 → 深度学习（LR、FM、DNN）
- **商业价值**：直接决定广告收入
- **相关内容**：[[机器学习]]、[[推荐系统]]

### 概念 3：用户定向

- **定义**：根据用户特征精准投放广告
- **定向方式**：人口属性、行为、上下文、lookalike
- **隐私挑战**：GDPR、CCPA、IDFA变更
- **相关内容**：[[DMP]]、[[用户画像]]

### 概念 4：eCPM

- **定义**：千次展示期望收益，衡量广告价值的核心指标
- **公式**：eCPM = CTR × CPC × 1000
- **优化目标**：提升CTR或CPC
- **相关内容**：[[广告收益优化]]
