# WeHan × 才聚江城

## 武汉人才留汉智能服务平台

---

## 一、产品概述

**WeHan 求职助手**是一款面向武汉高校毕业生的AI智能求职服务平台，采用"双端联动"架构，通过AI智能体技术为学生提供面试模拟、岗位匹配、政策咨询、心理疏导等一站式求职服务。

**核心定位**：不是招聘平台，而是"武汉大学生的就业AI顾问"

我们不做信息搬运，而是做**智能匹配 + 能力提升 + 决策辅助**。

| 项目 | 内容 |
|-----|------|
| 产品名称 | WeHan 求职助手（C端）/ 才聚江城（B端） |
| 当前版本 | Demo v1.0 |
| 目标用户 | 武汉80+高校、130万在校大学生 |

---

## 二、解决什么问题？

| 问题 | 现状 | 我们的解法 |
|-----|------|-----------|
| 学生找工作难 | 不知道自己适合什么岗位，面试没经验 | AI面试模拟 + 个性化评估报告 |
| 企业招人难 | 简历太多，筛选效率低 | AI预筛 + 匹配度评分 |
| 政府留人难 | 政策多但学生不知道 | 政策智能匹配 + 留汉数据可视化 |
| 学校就业率压力 | 缺乏有效的就业服务工具 | 就业看板 + 岗位推送 |

## 三、C端产品：WeHan 求职助手

### 3.1 产品定位

面向武汉高校求职学生的AI智能助手，提供求职全流程陪伴服务。

### 3.2 核心功能

#### AI 模拟面试

> **智能体核心能力**：自主决策 + 任务拆解 + 工具调用 + 目标执行

| 能力 | 说明 |
|-----|------|
| 目标理解 | 基于用户选择的岗位，智能理解面试考察目标 |
| 问题生成 | 自主生成针对性的面试问题 |
| 自主决策 | 根据回答实时调整提问方向和难度 |
| 综合评估 | 多维度分析生成评估报告 |

**面试流程**：
```
选择岗位 → 开始面试 → 逐题作答 → 实时评分 → 生成报告
```

**支持岗位**：前端开发工程师、后端开发工程师、AI算法工程师

#### 其他功能

| 功能 | 说明 |
|-----|------|
| 岗位查询 | 查询武汉地区热门岗位信息 |
| 政策查询 | 了解武汉人才政策、就业补贴、落户优惠 |
| 心理疏导 | 倾诉求职压力，获得情感支持 |
| 面试断点恢复 | 中途退出后可继续上次面试 |
| 快捷指令 | 4个快捷气泡一键触发功能 |

### 3.3 技术实现

| 项目 | 技术方案 |
|-----|---------|
| 智能体平台 | 扣子（Coze）+ 3个工作流 |
| 用户载体 | Chat SDK 网页嵌入 |
| AI能力 | 豆包大模型 + HTTP插件调用B端API |

### 3.4 用户价值

- **免费AI面试练习**：随时随地模拟真实面试
- **个性化评估报告**：了解自己的优势和不足
- **心理陪伴支持**：求职路上不孤单

## 四、B端产品：才聚江城管理平台

### 4.1 产品定位

面向企业、政府、学校、管理员的四端协同管理平台，实现人才留汉全链条数字化。

### 4.2 四端功能

#### 企业端

| 功能 | 说明 |
|-----|------|
| 首页概览 | 今日招聘数据、快捷操作入口 |
| 岗位管理 | 发布岗位、岗位上下架 |
| 人才库 | 投递列表、查看AI评估报告、候选人筛选 |
| 面试管理 | 面试安排、状态跟踪 |

#### 政府端

| 功能 | 说明 |
|-----|------|
| 留汉指数大屏 | 投递数、签约数、留汉率、企业参与数 |
| 趋势分析 | 月度/季度数据趋势、行业分布 |
| 政策管理 | 发布政策、政策触达追踪 |

#### 学校端

| 功能 | 说明 |
|-----|------|
| 就业看板 | 就业率、留汉率、专业去向分布 |
| 岗位推送 | 选择岗位推送给指定专业学生 |
| 学生管理 | 学生就业状态追踪 |

#### 管理员后台

| 功能 | 说明 |
|-----|------|
| 内容管理 | 岗位数据维护、政策信息维护 |
| 审核中心 | 企业入驻审核、学校入驻审核 |
| 用户管理 | 企业/学校/政府账号管理 |

### 4.3 技术实现

| 项目 | 技术方案 |
|-----|---------|
| 前端框架 | Next.js 16 + TypeScript |
| UI组件 | Ant Design 5 + Tailwind CSS |
| 后端框架 | Next.js API Routes + Prisma |
| 数据库 | PostgreSQL |
| 认证系统 | NextAuth.js（4角色RBAC） |

### 4.4 开放API

| 端点 | 方法 | 说明 |
|-----|------|------|
| `/api/open/jobs` | GET | 获取岗位列表 |
| `/api/open/policies` | GET | 获取政策列表 |
| `/api/open/interviews` | GET/POST | 查询/创建面试 |
| `/api/open/interviews/:id` | PATCH | 更新面试进度 |

## 六、OPC平台协作需求

我们希望通过入驻武汉市OPC线上开放社区，获得以下支持：

### 6.1 场景对接

| 对接方向 | 具体需求 |
|---------|---------|
| **高校就业中心** | 对接武汉各高校就业指导中心，将WeHan纳入就业服务工具体系 |
| **企业招聘渠道** | 对接武汉重点企业HR系统，实现岗位数据互通 |
| **政务数据** | 对接武汉人才政策库，实现政策精准推送 |
| **产业园区** | 对接东湖高新区、经开区等产业园区招聘需求 |

### 6.2 资源支持

| 支持类型 | 具体需求 |
|---------|---------|
| **算力支持** | AI面试并发扩展所需的普惠算力资源 |
| **推广渠道** | 面向全市政企单位、高校院所、百万大学生推广 |
| **线下空间** | 优先对接各区OPC线下社区场所资源 |

### 6.3 预期成效

通过OPC平台的多端协作，预计实现：

- **学生端**：求职效率提升30%，面试通过率提升20%
- **企业端**：招聘筛选效率提升50%，降低30%无效面试
- **政府端**：人才留汉率可视化追踪，政策触达率提升
- **学校端**：就业服务数字化，就业数据实时掌握
