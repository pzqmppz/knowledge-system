# 武汉人才留汉智能服务平台 - 产品设计文档

> 产品定位：连接武汉高校人才与本地企业的智能服务平台
> 文档创建：2026-02-27
> 状态：设计中

---

## 一、产品定位

### 1.1 核心定位
```
武汉人才留汉智能服务平台 ≠ 招聘平台
```

| 是 | 不是 |
|---|---|
| 信息聚合与推荐 | 招聘中介 |
| 职业规划顾问 | 简历代写服务 |
| 面试准备助手 | 面试官培训 |
| 武汉本地化服务 | 全国性招聘平台 |
| 人才留汉助推器 | 猎头服务 |

### 1.2 核心价值主张

> 帮助武汉大学生留在武汉，> 帮助武汉企业找到本地人才
> 帮助政府实现人才留汉政绩

### 1.3 利益相关方

| 对谁 | 价值 | 付费动机 |
|------|------|---------|
| **对政府** | 响应"人才留汉"政策，有政绩 | 有预算，有政绩需求 |
| **对学校** | 提高就业率指标 | 有就业率指标压力 |
| **对企业** | 获得本地人才渠道 | 有招聘需求，有预算 |
| **对学生** | 免费服务，有实际价值 | 免费，后续可增值服务 |
| **对创始人** | 获得启动资金 + 政府背书 + 后续商业化可能 | 出力换资源 |

## 二、用户流程

### 2.1 学生端

```
注册 → 完善信息(专业/方向) →
→ 获取岗位推荐 → AI简历诊断 → 面试模拟 → 获取内推
```

### 2.2 企业端（可选）

```
注册 → 发布校招信息 → 获取学生推荐 → 查看简历 → 发起邀约
```

## 三、核心功能模块

### 3.1 首页
- 岗位推荐流（基于专业和兴趣）
- 就业资讯（武汉本地政策、招聘会信息）
- AI助手入口
- 数据看板（本专业就业率、薪资分布）

### 3.2 智能顾问
- **简历诊断**：上传简历 → AI分析 → 优化建议
- **面试模拟**：选择岗位类型 → AI扮演面试官 → 反馈评分
- **职业规划**：描述背景 → AI推荐方向 → 路径规划
- **政策解读**：武汉人才政策问答

### 3.3 岗位聚合
- 本地企业校招信息
- 政府人才引进项目
- 实习岗位推荐
- 内推渠道

## 四、差异化价值

### 4.1 竞品对比

| 竞品 | 定位 | 优势 | 不足 |
|------|------|------|------|
| **智联招聘/前程无忧** | 全国性招聘平台 | 岗位多 | 缺乏武汉本地化 |
| **BOSS直聘** | 直聊招聘 | 互动快 | 侧重社招，校招信息分散 |
| **各高校就业网** | 校内就业信息 | 本校精准 | 信息孤岛 |
| **讯飞AI就业助手** | AI求职顾问 | 技术成熟 | 通用化，缺乏本地化 |
| **超级简历** | AI简历优化 | 简历功能专业 | 缺乏面试模拟 |
| **牛客网** | 面试题库+模拟 | 技术岗资源丰富 | 非技术岗较弱 |

### 4.2 本产品差异化

1. **武汉本地化**：聚焦武汉企业和高校
2. **产业结合**：对接武汉重点产业（汽车、医药、光电子）
3. **政策解读**：人才留汉政策智能问答
4. **AI深度辅助**：不只是信息展示，而是智能顾问

### 4.3 核心差异点

> "武汉大学生的就业AI顾问"

- **武汉本地化**：聚焦武汉企业和高校
- **产业对接**：与武汉965产业体系结合
- **政策解读**：人才留汉政策智能问答
- **AI深度服务**：简历诊断+面试模拟+职业规划

### 4.4 突破性差异：C端主导的反馈机制

> 核心洞察：传统招聘平台都是B端付费主导，C端用户没有话语权

**传统平台的问题**：
```
B端付费 → 平台服务B端 → C端被动接受 → 体验差也无法反馈
```

**我们的突破**：
```
┌─────────────────────────────────────────────────────────────┐
│              C端主导的向上反馈修复机制                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  C端用户 ──→ 投递岗位 ──→ 体验反馈 ──→ 平台介入             │
│                               │                             │
│                               ├── 岗位评分（1-5星）         │
│                               ├── 企业评分（1-5星）         │
│                               └── 问题标签                  │
│                                   ├── 无反馈/已读不回       │
│                                   ├── JD与实际不符          │
│                                   ├── 面试体验差            │
│                                   └── 薪资虚标              │
│                                                             │
│  触发机制：                                                  │
│  ├── 某岗位/企业评分过低 → 平台主动沟通企业                 │
│  ├── 问题未改善 → 降权/下架                                 │
│  └── 严重问题 → 加入黑名单                                  │
│                                                             │
│  价值：C端用户第一次拥有对招聘质量的"投票权"                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**与传统平台对比**：

| 维度 | 传统招聘平台 | 本产品 |
|------|-------------|--------|
| 谁是付费方 | B端（企业） | 政府采购+增值服务 |
| 谁被服务 | B端优先 | C端（学生）优先 |
| 反馈机制 | 无/C端投诉无门 | C端评分→触发平台介入 |
| 岗位质量 | 企业自己写，无人监管 | C端反馈+平台审核 |
| 劣质企业 | 付费就能继续 | 评分低→降权/下架 |

## 五、数据来源策略

| 来源 | 说明 | 合规性 |
|------|------|--------|
| **高校就业网** | 武汉80+高校就业信息聚合 | 公开信息 |
| **企业官网** | 本地重点企业校招页面 | 公开信息 |
| **政府平台** | 人才政策、招聘会信息 | 公开信息 |
| **企业自主发布** | 企业HR入驻发布岗位 | 平台审核 |

### 5.1 数据获取原则

> 核心原则：只使用**公开可获取**的信息，不爬取招聘平台数据

- **可以做的**：聚合高校就业网、企业官网的公开校招信息
- **不做的**：爬取智联、BOSS直聘等招聘平台的数据
- **鼓励的**：企业HR主动入驻发布，信息更准确及时

### 5.2 岗位异议反馈机制

每个岗位页面底部设置「信息有误/已失效？」反馈入口：

```
用户反馈 ──→ 选择问题类型 ──→ 平台审核 ──→ 处理
              │
              ├── 岗位已招满/失效 ──→ 下架
              ├── 联系方式错误 ──→ 联系企业确认/下架
              ├── JD与实际不符 ──→ 联系企业/下架
              └── 其他问题 ──→ 人工审核
```

**处理时效**：
- 紧急问题（已失效）：24小时内下架
- 普通问题（信息有误）：3个工作日内处理

### 5.3 冷启动策略

1. **数据预填充**：聚合公开校招信息
2. **种子用户**：与1-2个高校就业中心合作
3. **政策背书**：强调"服务武汉大学生留汉就业"
4. **内推机制**：鼓励企业HR入驻发布

## 九、产品形态参考

### 9.1 参考产品：Oboe AI

> 来源：https://oboe.com/
> 详细拆解：[[Oboe-AI生成课程产品拆解报告]](../拆解报告/产品/Oboe-AI生成课程产品拆解报告.md)

**可借鉴**：
- 分章节引导式体验（求职流程：简历→面试→offer）
- 结构化学习指南模板
- 测验反馈机制（模拟面试评分）

**不借鉴**：
- 章节式讲解（限制探索自由度）
- 课程形态（求职不是学习课程）

### 9.2 产品形态思考

| 产品形态 | 适合场景 | 是否适合本产品 |
|---------|---------|---------------|
| **引导型**（章节+Next） | 教学、培训、求职准备 | ✅ 适合 |
| **探索型**（自由跳转） | 研究、发现、创新 | ❌ 不适合 |

**结论**：本产品适合**引导型**产品形态，因为：
- 求职有明确流程：简历→投递→面试→offer
- 用户需要按步骤完成准备
- 有明确的终点（找到工作）

### 9.3 C端产品快速落地方案

> 核心思路：借助Code平台智能体，快速搭建对话流产品

#### AI面试模拟完整流程

```
┌─────────────────────────────────────────────────────────────┐
│                    AI面试模拟对话流                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 简历解析                                           │
│  ├── 用户上传简历/输入信息                                  │
│  └── AI解析：技能、经历、求职意向                           │
│                                                             │
│  Step 2: 岗位匹配                                           │
│  ├── AI推荐匹配岗位                                         │
│  └── 用户选择感兴趣的岗位                                   │
│                                                             │
│  Step 3: 面试大纲生成（参考Oboe AI模式）                    │
│  ├── AI根据岗位JD生成10道面试题                             │
│  ├── 用户可浏览题目列表                                     │
│  └── 用户点击「开始模拟面试」进入对话流                     │
│                                                             │
│  Step 4: 对话流面试（流媒体传输）                           │
│  ├── AI抛出第一个问题（语音/文字）                          │
│  ├── 用户回答（语音/文字）                                  │
│  ├── AI根据回答：                                           │
│  │   ├── 追问细节（深入考察）                               │
│  │   ├── 跳转下一题（已充分了解）                           │
│  │   └── 拓展新问题（基于回答衍生）                         │
│  ├── 实际问题数：可能从10道拓展到20道                       │
│  └── 用户可随时暂停/继续                                    │
│                                                             │
│  Step 5: 面试评估报告                                       │
│  ├── 综合评分（0-100分）                                    │
│  ├── 各维度评分：表达能力、专业能力、逻辑思维...            │
│  ├── 优秀回答回顾                                           │
│  ├── 待改进点 + 改进建议                                    │
│  └── 推荐学习资源                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 技术实现要点

| 模块 | 实现方式 | 说明 |
|------|---------|------|
| **对话流引擎** | Code平台智能体 | 快速搭建，无需从零开发 |
| **流媒体传输** | WebSocket/SSE | 实时对话体验 |
| **语音识别** | ASR接口 | 用户语音回答转文字 |
| **语音合成** | TTS接口 | AI问题语音播报 |
| **评估算法** | LLM+结构化prompt | 多维度评分+建议生成 |

## 十一、产品优化建议（来自外部反馈）

### 11.1 政策一键兑现

> 核心洞察：政策多而散，学生不知道、不会用、用不上

```
┌─────────────────────────────────────────────────────────────┐
│                    政策一键兑现流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  学生信息 ──→ AI分析 ──→ 可享受政策列表 ──→ 一键申请        │
│     │                         │                             │
│     │                         ├── 租房补贴                  │
│     ├── 学历                  ├── 就业奖励金                │
│     ├── 专业                  ├── 落户政策                  │
│     ├── 意向区域              └── 创业扶持                  │
│     └── 就业状态                                           │
│                                                             │
│  价值：把"政策查询"变成"政策到账"                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**实现要点**：
- 对接政府政策数据库
- 自动匹配学生条件
- 生成申请材料清单
- 追踪申请进度

### 11.2 实习前置化与学分化

> 核心洞察：毕业才找工作的模式太落后，实习应该成为"课程"

| 传统模式 | 优化模式 |
|---------|---------|
| 大四才开始实习 | 大二/大三就开始 |
| 实习与学分脱钩 | 实习=选修课学分 |
| 企业与学校分离 | 校企联合培养 |

**实现方式**：
```
平台功能：
├── 实习岗位课程化（带学分标注）
├── 校企联合培养计划展示
├── 实习过程追踪与评价
└── 学分认定对接学校系统
```

**价值**：
- 学生：早接触职场，明确方向
- 企业：提前锁定人才，降低招聘成本
- 学校：提高实习质量，就业率提升

### 11.3 企业信任背书

> 核心洞察：学生怕踩坑，企业怕招不到人——需要信任桥梁

```
┌─────────────────────────────────────────────────────────────┐
│                    企业信任背书机制                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  企业入驻 ──→ 平台审核 ──→ 背书标签 ──→ 学生可见           │
│                 │              │                            │
│                 │              ├── ✓ 官方认证企业           │
│                 │              ├── ✓ 无劳动纠纷记录         │
│                 │              ├── ✓ 留汉就业优秀雇主       │
│                 │              └── ✓ 校企合作单位           │
│                 │                                           │
│                 └── 营业执照、社保缴纳、投诉记录等          │
│                                                             │
│  反向机制：学生评价 + 投诉通道 + 黑名单制度                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**标签体系**：
| 标签 | 获取条件 | 价值 |
|------|---------|------|
| 官方认证 | 营业执照+实地审核 | 基础信任 |
| 留汉优秀雇主 | 连续2年留汉率>50% | 政策倾斜 |
| 校企合作 | 与武汉高校有正式合作 | 学生优先 |

### 11.4 青年人才驿站联动

> 核心洞察：非武汉生源来汉找工作，住哪儿是第一个问题

**功能设计**：
```
平台集成：
├── 青年人才驿站地图（免费7-14天住宿）
├── 在线申请与资格审核
├── 入住体验评价
└── 驿站周边生活指南
```

**联动价值**：
- 政府驿站 → 获得精准用户
- 学生 → 解决住宿焦虑
- 平台 → 增加独家功能点

### 11.5 解决"隐形门槛"

> 核心洞察：招聘中的隐形歧视（学历、性别、年龄、地域）是痛点

**透明化机制**：
| 功能 | 实现方式 | 价值 |
|------|---------|------|
| 薪资透明 | 要求企业标注薪资范围 | 避免浪费时间 |
| 要求透明 | 明确学历/经验要求 | 减少无效投递 |
| 反馈透明 | 投递后必须有反馈 | 提升体验 |
| 歧视举报 | 隐形歧视举报通道 | 净化环境 |

**AI辅助**：
- 简历匿名化处理（去除姓名、性别、照片）
- 匹配度预测（投递前告知成功率）
- 面试体验反馈收集

### 11.6 四端深度运营建议（来自外部反馈）

> 核心洞察：让平台真正"活"起来，需要四端联动运营

#### 学生端：从"找工作"到"规划未来"

| 功能 | 说明 | 价值 |
|------|------|------|
| **职业性格测评+岗位匹配** | 对接武汉本土企业用人偏好，生成"武汉就业适配度报告" | 让学生知道自己适合光谷互联网还是沌口制造业 |
| **"留汉学长学姐"带路** | 邀请近2年留汉工作的毕业生入驻，分享真实薪资、租房经验、职场避坑 | 比官方宣传更有说服力 |
| **"试岗体验"机制** | 和企业合作，提供3-5天短期实训/开放日 | 学生先体验再决定，降低试错成本 |

#### 企业端：从"招人难"到"精准画像"

| 功能 | 说明 | 价值 |
|------|------|------|
| **动态人才画像系统** | 根据学校专业设置、学生实习数据，生成"高校人才供给热力图" | 告诉企业哪些学校有对口生源 |
| **批量面试/双选会自动化** | 一键筛选简历、安排线上面试、生成录用意向书 | 减少HR重复劳动 |
| **"留汉补贴计算器"** | 自动匹配武汉大学生就业补贴、社保减免、租房优惠等政策 | 让企业算清楚"招一个毕业生能省多少钱" |

#### 学校端：从"推就业"到"数据闭环"

| 功能 | 说明 | 价值 |
|------|------|------|
| **就业质量实时看板** | 哪个专业学生去了哪些企业、薪资多少、离职率如何 | 给学校调整专业设置提供数据支撑 |
| **校企合作"配对"功能** | 学校发布专业/毕业生/实习能力，企业发布需求，系统自动推荐配对 | 提高校企对接效率 |
| **就业指导资源库** | 武汉重点企业HR的面试技巧讲座视频、行业趋势报告 | 学校可直接引用到就业指导课 |

#### 政府端：可视化决策支持

| 功能 | 说明 | 价值 |
|------|------|------|
| **"留汉指数"大屏** | 实时显示大学生投递简历数、签约数、企业参与数、热门行业/岗位分布 | 政绩可视化 |
| **政策触达追踪** | 哪些学生领了租房补贴？哪些企业申请了社保补贴？资金流向哪里？ | 让每一分钱花得明明白白 |

### 11.7 三阶段推进与竞争壁垒（来自外部反馈）

> 核心洞察：分阶段推进，建立数据闭环壁垒

#### 三阶段产品规划

| 阶段 | 周期 | 核心目标 |
|------|------|---------|
| A | 6-12月 | 做"留汉就业闭环"最小可用版 |
| B | 12-24月 | 做"人才留汉经营"能力 |
| C | 24月+ | 产业人才生态平台 |

**阶段A关键动作**：
- 统一岗位标签体系（专业/技能/薪资/通勤/应届友好度）
- 统一人才画像（课程项目、竞赛、实习、作品集、技能自测）
- 统一就业结果回传（录用/到岗/转正）

**核心指标**：岗位有效率、投递转化率、留汉率、校企复用率

#### 商业模式：政府主导 + 市场化增值

| 收入来源 | 内容 |
|---------|------|
| 政府侧 | 平台建设、数据治理、就业/人才留存绩效服务 |
| 企业侧 | 精准招聘包、校企合作包、雇主品牌包 |
| 生态侧 | 人才公寓、租房、培训、测评等合作分成 |

#### 用户体验：三端共享同一份事实

| 端 | 核心体验 |
|---|---------|
| 学生端 | 一屏看懂城市机会 + 个人匹配建议 |
| 企业端 | JD自动结构化、候选人排序、招聘漏斗 |
| 学校端 | 专业就业看板、就业核验自动化 |

#### 竞争壁垒

| 壁垒 | 说明 |
|------|------|
| **独有数据闭环** | 在校→入职→留汉全链路数据 |
| **政策与公共服务直连** | 官方渠道独家接入 |
| **产业链运营能力** | 与武汉965产业深度绑定 |
| **政府背书可信标识** | 官方认证身份 |
| **标准体系** | 岗位标签、技能标准制定者 |
