## 四、关键决策

### 4.1 平台选择

| 决策点 | 选择 | 理由 |
|-------|------|------|
| C端载体 | 扣子智能体 + Chat SDK | 时间紧迫，快速上线 |
| B端框架 | Next.js 16 | 全栈开发效率高 |
| 部署方式 | 腾讯云 + 宝塔 | 成本低，易维护 |

### 4.2 产品策略

| 决策点 | 选择 | 理由 |
|-------|------|------|
| 评分机制 | 建议不拦截 | AI没有资格拦截学生的机会 |
| C端独立性 | 自定义岗位功能 | 不依赖B端数据也能完整体验 |
| 四端联动 | 学生/企业/政府/学校 | 对接通知要求，体现平台价值 |

### 4.3 品牌策略

## 六、经验总结

### 做对了什么

1. **快速决策**：平台选择（扣子+Next.js）果断，没有纠结
2. **MVP思维**：聚焦核心功能，不追求完美
3. **文档先行**：PRD完整，后续开发有章可循
4. **身份定位**：OPC超级个体身份与征集主题高度契合

### 可以改进

1. **时间管理**：截止前3天才开始，压力较大
2. **B端UI**：时间不足，部分页面待完善
3. **测试覆盖**：匆忙上线，测试不够充分

### 方法论沉淀

```
OPC产品开发流程：
1. 看通知 → 抓重点（审核人员看重什么）
2. 定方向 → 选赛道（与政策契合）
3. 写PRD → 明确范围（MVP思维）
4. 选技术 → 快速落地（成熟方案优先）
5. 做材料 → 精准表达（对齐审核标准）
6. 交申报 → 完成闭环
```
