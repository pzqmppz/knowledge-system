# 第一部分：方案选择

## 🎯 本地公网 IP 的常见误区

### ⚠️ 重要澄清

```
错误理解：
  "使用本地 IP 进行公网暴露" ❌

正确理解：
  "申请公网 IP + 端口映射" 或 "使用隧道服务" ✅

示例：
  本地 IP：192.168.1.100  （私有 IP，无法直接从外网访问）
  公网 IPv4：123.45.67.89    （运营商分配，可以从外网访问）
  公网 IPv6：2001:db8::1     （运营商分配，兼容性有限）

路由器的作用：
  公网 IP ←→ 路由器 ←→ 本地 IP (192.168.1.100)
                      端口映射
```

### 如何检查是否有公网 IP

```powershell
# 方法 1：访问 IP 查询网站
# https://ip.sb
# https://www.ip138.com
# https://ifconfig.me

# 方法 2：命令行查询
curl ip.sb

# 方法 3：PowerShell 脚本
 Invoke-RestMethod -Uri "http://ip.sb"
```

**判断标准：**
- 如果是 `10.x.x.x` / `192.168.x.x` / `172.16-31.x.x` → 没有公网 IP
- 如果是其他地址 → 可能有公网 IP（但可能还在 NAT 后面）

## 📞 申请公网 IP 流程

### 第一步：确认是否已有公网 IP

```powershell
# 运行网络检查脚本（已创建）
cd "e:\文档\code\知识体系\AI工具"
.\检查网络环境.ps1
```

### 第二步：联系运营商

| 运营商 | 客服电话 | 申请方式 |
|--------|---------|---------|
| **中国电信** | 10000 | 拨打客服 / 营业厅 |
| **中国移动** | 10086 | 拨打客服 / 营业厅 |
| **中国联通** | 10010 | 拨打客服 / 营业厅 |

### 话术参考

```
"你好，我家里需要用 NAS/智能家居，需要申请公网 IP，
 请问我的宽带支持开通吗？需要什么手续？"

注意事项：
  1. 企业宽带更容易申请 IPv4
  2. 家庭宽带部分地区只支持 IPv6
  3. 可能需要额外费用（通常免费）
  4. 开通后需要重启光猫

⚠️ 重要：申请时问清楚是 IPv4 还是 IPv6
```

### 第三步：验证公网 IP

```bash
# 1. 查询外网 IP
curl ip.sb

# 2. 判断 IP 类型
# IPv4 示例：123.45.67.89
# IPv6 示例：2001:db8::1

# 3. 在手机 4G/5G 网络下，尝试访问：
ping 你的公网IP

# 4. 如果能 ping 通，说明有公网 IP
```

### 第四步：确认 IP 类型

| IP 类型 | 特点 | 适用场景 |
|--------|------|---------|
| **静态公网 IP (IPv4)** | 固定不变 | ✅ 推荐，方便配置域名 |
| **动态公网 IP (IPv4)** | 重启路由器会变 | ✅ 需要 DDNS 服务 |
| **静态公网 IP (IPv6)** | 固定不变 | ⚠️ 部分网络无法访问 |
| **动态公网 IP (IPv6)** | 重启路由器会变 | ⚠️ 兼容性问题 |

**查看你的 IP 类型：**
```bash
# 1. 记录当前 IP
curl ip.sb > current-ip.txt

# 2. 重启光猫/路由器

# 3. 再次查询
curl ip.sb

# 4. 对比两次的 IP
# 如果相同 → 静态 IP
# 如果不同 → 动态 IP
```

### ⚠️ IPv6 特别说明

**如果你只有 IPv6 公网 IP：**

```
现状：
  ✅ 有公网 IP（IPv6）
  ❌ 没有 IPv4 公网 IP

问题：
  ⚠️ IPv6 兼容性问题：
     - 部分移动网络不支持 IPv6
     - 老旧设备/系统不支持 IPv6
     - 某些企业网络禁用 IPv6

  ⚠️ 访问限制：
     - 只有支持 IPv6 的网络才能访问
     - 你的朋友可能无法访问你的服务

  ⚠️ DDNS 限制：
     - 部分 DDNS 服务不支持 IPv6
     - 路由器端口映射可能不支持 IPv6

推荐方案：
  ✅ 使用 Cloudflare Tunnel（方案 C）
     - 不需要公网 IP
     - 自动处理 IPv4/IPv6 转换
     - 所有人都能访问
     - 免费且稳定
     - 5 分钟配置完成
```

# 第二部分：具体实施

## 方案 A：DDNS + 端口映射（仅 IPv4）

> ⚠️ **仅适用于 IPv4 公网 IP**
>
> 如果你是 IPv6 或没有公网 IP，请使用方案 C

### 架构

```
互联网
  ↓
公网 IPv4: 123.45.67.89
  ↓
路由器 (端口映射)
  ↓
本地设备
  ├─ Dify: 192.168.1.100:8090
  ├─ LocalAI: 192.168.1.100:8080
  └─ Ollama: 192.168.1.100:11434
```

### 步骤 1：配置 DDNS

**免费 DDNS 服务：**

| 服务商 | 免费域名 | IPv6 支持 | 特点 |
|--------|---------|----------|------|
| **DuckDNS** | yourdomain.duckdns.org | ✅ 支持 | 简单易用 |
| **No-IP** | yourname.ddns.net | ✅ 支持 | 老牌，稳定 |
| **花生壳** | 随机域名 | ⚠️ 有限 | 国内服务商 |

**以 DuckDNS 为例：**

```bash
# 1. 注册 DuckDNS
访问：https://www.duckdns.org
注册并选择一个子域名

# 2. 配置路由器 DDNS
登录路由器管理界面
找到 "DDNS" 或 "动态 DNS" 设置
选择 DuckDNS
输入你的域名和 Token
保存并启用

# 3. 验证
ping yourdomain.duckdns.org
# 应该解析到你的公网 IP
```

### 步骤 2：配置路由器端口映射

**登录路由器管理界面**

```
常见路由器地址：
- TP-Link: http://192.168.1.1
- 小米: http://192.168.31.1
- 华硕: http://router.asus.com
```

**配置端口映射规则**

| 服务名称 | 外部端口 | 内部 IP | 内部端口 | 协议 |
|---------|---------|---------|---------|------|
| Dify | 8090 | 192.168.1.100 | 8090 | TCP |
| LocalAI | 8080 | 192.168.1.100 | 8080 | TCP |

**配置示例（TP-Link 路由器）：**

```
1. 登录路由器
2. "转发规则" → "虚拟服务器"
3. 添加新条目：
   - 服务端口：8090
   - 内部端口：8090
   - IP 地址：192.168.1.100
   - 协议：TCP
   - 状态：生效
4. 保存
```

### 步骤 3：测试公网访问

```bash
# 1. 在手机 4G 网络下测试
curl http://yourdomain.duckdns.org:8090

# 2. 浏览器访问 Dify
http://yourdomain.duckdns.org:8090/console
```

## 方案 B：frp + 云服务器

### 架构

```
互联网
  ↓
云服务器 (公网 IPv4 + frps + Nginx)
  ↓
frp 隧道（加密）
  ↓
本地设备 (frpc)
  ├─ Dify: localhost:8090
  ├─ LocalAI: localhost:8080
  └─ Ollama: localhost:11434
```

### 优势

```
✅ 不需要公网 IP
✅ 不需要申请运营商
✅ 不需要路由器端口映射
✅ 加密传输
✅ 可以绑定自己的域名
✅ 稳定可控
✅ 完美支持 IPv4/IPv6 转换
✅ 可加 API Key 鉴权
```

### 详细配置

#### 1. 云服务器配置 frps

**安装 frp：**
```bash
# SSH 连接到云服务器
ssh root@your-server-ip

# 下载 frp
wget https://github.com/fatedier/frp/releases/download/v0.54.0/frp_0.54.0_linux_amd64.tar.gz
tar -xzf frp_0.54.0_linux_amd64.tar.gz
cd frp_0.54.0_linux_amd64
```

**创建配置文件 `frps.toml`：**
```toml
# frps.toml
bindPort = 7000

# Dashboard（可选）
webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "你的强密码_至少20位"

# 认证
auth.method = "token"
auth.token = "你_64位_随机_超级_复杂_令牌"
```

**生成安全令牌：**
```bash
openssl rand -hex 32
```

**启动 frps：**
```bash
# 测试运行
./frps -c frps.toml

# 设置为系统服务
sudo nano /etc/systemd/system/frps.service
```

**创建服务文件：**
```ini
[Unit]
Description = frp server
After = network.target

[Service]
Type = simple
User = root
Restart = on-failure
RestartSec = 5s
ExecStart = /root/frp_0.54.0_linux_amd64/frps -c /root/frp_0.54.0_linux_amd64/frps.toml

[Install]
WantedBy = multi-user.target
```

**启动服务：**
```bash
sudo systemctl daemon-reload
sudo systemctl enable frps
sudo systemctl start frps
sudo systemctl status frps
```

**开放端口：**
```bash
# 在云服务器控制台安全组开放：
# - TCP 7000（frp 通信）
# - TCP 6000（映射给 LocalAI）
# - TCP 7500（Dashboard，可选）
```

#### 2. 本地 Windows 配置 frpc

**下载 frp 客户端：**

下载地址：https://github.com/fatedier/frp/releases/download/v0.54.0/frp_0.54.0_windows_amd64.zip

解压到：`C:\frp\`

**创建配置文件 `frpc.toml`：**
```toml
# frpc.toml
serverAddr = "你的云服务器IP"
serverPort = 7000

# 认证（必须与服务器一致）
auth.method = "token"
auth.token = "你_64位_随机_超级_复杂_令牌"

# Dify 服务
[[proxies]]
name = "dify-web"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8090      # Dify 端口
remotePort = 6000     # 云服务器暴露端口

# LocalAI 服务（可选）
# [[proxies]]
# name = "localai"
# type = "tcp"
# localIP = "127.0.0.1"
# localPort = 8080
# remotePort = 6001
```

**测试运行：**
```cmd
cd C:\frp
frpc -c frpc.toml
```

**设置为 Windows 服务：**

下载 NSSM：https://nssm.cc/download

```cmd
# 以管理员身份运行
nssm install frpc C:\frp\frpc.exe
nssm set frpc AppParameters -c C:\frp\frpc.toml
nssm set frpc DisplayName FRP Client
nssm set frpc Description 连接本地服务到云服务器
nssm set frpc Start SERVICE_AUTO_START
nssm start frpc
```

#### 3. 云服务器 Nginx 配置

**创建配置文件：**
```bash
sudo nano /etc/nginx/sites-available/dify-api
```

**Nginx 配置：**
```nginx
# /etc/nginx/sites-available/dify-api

# 生成 API Key
openssl rand -hex 16

map $http_authorization $is_valid_key {
    default 0;
    "Bearer sk-你生成的_32位_密钥" 1;
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name dify.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl http2;
    server_name dify.yourdomain.com;

    # SSL 证书
    ssl_certificate /etc/letsencrypt/live/dify.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/dify.yourdomain.com/privkey.pem;

    # SSL 优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 限流保护
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;

    location / {
        # 限流
        limit_req zone=api_limit burst=20 nodelay;

        # API Key 鉴权
        if ($is_valid_key = 0) {
            return 401 '{"error":"Unauthorized","message":"Invalid or missing API key"}';
        }

        # CORS（如果需要）
        add_header 'Access-Control-Allow-Origin' '*' always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS' always;
        add_header 'Access-Control-Allow-Headers' 'Authorization, Content-Type' always;

        if ($request_method = 'OPTIONS') {
            return 204;
        }

        # 代理到 frp
        proxy_pass http://127.0.0.1:6000;

        # 代理头
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 流式响应支持
        proxy_buffering off;
        proxy_cache off;

        # 超时设置
        proxy_connect_timeout 300s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }

    # 健康检查（无需鉴权）
    location /health {
        access_log off;
        return 200 '{"status":"ok","service":"Dify"}';
    }
}
```

**启用配置：**
```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/dify-api /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

**申请 SSL 证书：**
```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d dify.yourdomain.com

# 自动续期（已自动设置）
sudo certbot renew --dry-run
```

## 方案 C：Cloudflare Tunnel（推荐）

### 架构

```
互联网
  ↓
Cloudflare CDN (自动 IPv4/IPv6)
  ↓
Cloudflare Tunnel (加密)
  ↓
本地设备 (cloudflared)
  ├─ Dify: localhost:8090
  ├─ LocalAI: localhost:8080
  └─ 其他服务: localhost:xxxx
```

### 优势

```
✅ 完全免费
✅ 不需要云服务器
✅ 不需要公网 IP（IPv4 或 IPv6 都不需要）
✅ 自动处理 IPv4/IPv6 转换
✅ 自动 HTTPS
✅ 全球 CDN 加速
✅ DDoS 防护
✅ 所有人都能访问（完美兼容性）
```

### 适用场景

| 场景 | 是否适用 |
|------|---------|
| 有 IPv4 公网 IP | ✅ 可用 |
| **只有 IPv6 公网 IP** | ✅ **强烈推荐** |
| 没有公网 IP | ✅ **强烈推荐** |
| 想要 HTTPS | ✅ 原生支持 |
| 免费方案 | ✅ 完全免费 |

### 详细配置

#### 1. 安装 cloudflared

**Windows：**
```cmd
# 下载最新版本
# https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.exe

# 重命名为 cloudflared.exe
# 添加到 PATH 或放到 C:\Windows\System32\
```

#### 2. 登录 Cloudflare

```cmd
# 登录（会打开浏览器授权）
cloudflared tunnel login
```

1. 浏览器会自动打开 Cloudflare 登录页面
2. 登录你的 Cloudflare 账号（免费注册）
3. 选择你要使用的域名
4. 授权完成后返回命令行

#### 3. 创建隧道

```cmd
# 创建隧道
cloudflared tunnel create dify-tunnel

# 记录返回的信息：
# Tunnel ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
# Credentials file 已保存
```

#### 4. 创建配置文件

**创建配置文件 `C:\Users\你的用户名\.cloudflared\config.yml`：**

```yaml
# cloudflared 配置
tunnel: <你的-Tunnel-ID>
credentials-file: C:\Users\你的用户名\.cloudflared\<你的-Tunnel-ID>.json

# 日志
loglevel: info
#logfile: C:\Users\你的用户名\.cloudflared\logs\cloudflared.log

ingress:
  # Dify 主服务
  - hostname: dify.yourdomain.com
    service: http://localhost:8090

  # API 服务（可选，用于鉴权）
  - hostname: api-dify.yourdomain.com
    service: http://localhost:8090

  # LocalAI（可选）
  - hostname: localai.yourdomain.com
    service: http://localhost:8080

  # 健康检查
  - hostname: dify.yourdomain.com
    path: /^health$
    service: http_status:200

  # 其他请求返回 404
  - service: http_status:404
```

#### 5. 配置 DNS

**方式 1：使用命令自动配置**

```cmd
# 添加 DNS 记录
cloudflared tunnel route dns dify-tunnel dify.yourdomain.com
cloudflared tunnel route dns dify-tunnel api-dify.yourdomain.com
cloudflared tunnel route dns dify-tunnel localai.yourdomain.com
```

**方式 2：手动在 Cloudflare 控制台配置**

1. 登录 Cloudflare 控制台
2. 选择你的域名
3. 进入 DNS 设置
4. 添加 CNAME 记录：
   - 类型：CNAME
   - 名称：dify
   - 目标：`<你的-Tunnel-ID>.cfargotunnel.com`
   - 代理状态：已启用（橙色云朵）

#### 6. 测试运行

```cmd
# 测试运行
cloudflared tunnel run dify-tunnel

# 保持运行...
```

#### 7. 设置为 Windows 服务（开机自启动）

```cmd
# 以管理员身份运行
cloudflared service install
```

| **速度** | 快 | 快 | 很快（CDN） |
| **安全性** | 需自己加固 | 高（可加鉴权） | 高（Cloudflare） |

## 🎯 总结建议

| 你的情况 | 推荐方案 | 预计时间 | 成本 |
|---------|---------|---------|------|
| 只有 IPv6 公网 IP | **Cloudflare Tunnel** | 5 分钟 | 免费 |

| **没有公网 IP** | **Cloudflare Tunnel** | 5 分钟 | 免费 |
