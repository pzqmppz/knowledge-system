# 网页截图 API - 需求讨论稿

> **项目类型**: 个人工具 / 内部服务
> **创建日期**: 2026-03-05
> **参考产品**: [ApiFlash](https://apiflash.com/)

---

## 一、项目背景

### 1.1 为什么需要这个项目

- **成本考虑**: ApiFlash 等商业服务按量收费，个人使用成本较高
- **数据隐私**: 截图内容可能包含敏感信息，不希望经过第三方服务
- **可控性**: 自建服务可以灵活定制功能和参数
- **学习价值**: 了解 Headless Browser、API 设计等技术栈

### 1.2 参考产品分析

**ApiFlash** 核心功能：
- 基于 Chrome 渲染的截图
- 全页/可视区域截图
- 移动端/响应式视口
- 多种图片格式 (PNG/JPEG/WebP)
- 自动等待页面加载
- 广告屏蔽、Cookie 横幅隐藏

**我们的目标**: 实现核心功能，满足个人使用即可

## 二、核心功能需求

### 2.1 MVP (最小可行产品)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| URL 截图 | 输入 URL，返回截图 | P0 |
| 图片格式 | 支持 PNG/JPEG | P0 |
| 视口设置 | 可自定义宽度/高度 | P1 |
| 全页截图 | 可选截取整个页面 | P1 |
| API 鉴权 | 简单的 API Key 验证 | P1 |

### 2.2 基础 API 设计

```
GET /api/screenshot
```

**请求参数**:
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| url | string | 是 | - | 目标网页 URL |
| width | number | 否 | 1920 | 视口宽度 |
| height | number | 否 | 1080 | 视口高度 |
| full_page | boolean | 否 | false | 是否全页截图 |
| format | string | 否 | png | 图片格式: png/jpeg/webp |
| quality | number | 否 | 90 | JPEG 质量 (1-100) |
| wait_for | number | 否 | 0 | 等待时间 (毫秒) |

**响应**: 图片二进制数据

## 三、技术选型

### 3.1 技术栈对比

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Puppeteer** | 成熟稳定，文档完善 | 仅支持 Chrome | ⭐⭐⭐⭐⭐ |
| **Playwright** | 跨浏览器，API 更好 | 体积稍大 | ⭐⭐⭐⭐ |
| **Chrome Lambda (无头浏览器)** | - | 维护成本高 | ⭐⭐ |

### 3.2 推荐方案: Puppeteer + Node.js

```javascript
// 核心依赖
{
  "puppeteer": "^23.x",      // 浏览器自动化
  "express": "^4.x",         // Web 框架
  "sharp": "^0.33.x"         // 图片处理 (可选)
}
```

### 3.3 部署环境

| 要求 | 建议 |
|------|------|
| 操作系统 | Ubuntu 20.04+ |
| 内存 | 最低 2GB (推荐 4GB+) |
| CPU | 2 核及以上 |
| 存储 | 10GB+ (Chrome 需要空间) |
| 运行时 | Node.js 18+ |

## 四、架构设计

```
┌─────────────────────────────────────────────────────┐
│                    Nginx (可选)                      │
│              / 反向代理 + SSL + 缓存                  │
└─────────────────────┬───────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────┐
│               Express API Server                     │
│  ┌─────────────────────────────────────────────┐   │
│  │  鉴权中间件 → 参数校验 → 速率限制             │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────┬───────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────┐
│              Puppeteer Browser Pool                  │
│  ┌─────────────────────────────────────────────┐   │
│  │  复用浏览器实例，避免频繁启动                 │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 4.1 关键设计点

1. **浏览器池管理**: 复用实例，减少启动开销
2. **超时控制**: 避免页面卡死导致资源泄漏
3. **错误处理**: 自动重启崩溃的浏览器
4. **日志记录**: 记录请求和错误信息
5. **健康检查**: 提供服务状态端点
