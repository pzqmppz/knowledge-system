## 系统架构

### 事件驱动架构

```
监听层 (Webhook)
    ↓
路由层 (格式识别与分发)
    ↓
处理层 (Web车间 / PDF车间 / Data车间)
    ↓
认知层 (LLM 处理)
    ↓
回填层 (Notion 写入)
```

---

## 核心模块

### 1. 内容提取器 (Content Refinery)

**文件**：`lib/content-refinery.ts`

多层提取策略：
1. 标准解析
2. XML 模式 (xmlMode)
3. 正则兜底

```typescript
// 多层回退策略
async extractContent(epub: EpubData): Promise<Chapter[]> {
  try {
    return await this.standardExtract(epub);
  } catch {
    try {
      return await this.xmlModeExtract(epub);
    } catch {
      return await this.regexExtract(epub);
    }
  }
}
```

### 2. AI 处理器 (AI Processor)

**文件**：`lib/ai-processor.ts`

双模型支持：
- **GLM-4.7-Flash**：快速处理
- **DeepSeek-V3**：深度分析

超时控制：240s

### 3. Notion 适配器

**文件**：`lib/notion-adapter.ts`

功能：
- 数据库写入
- 状态更新
- 统计查询

### 4. 重试与熔断器

**文件**：`lib/retry.ts`, `lib/circuit-breaker.ts`

- 指数退避 + Jitter
- 熔断器防止级联故障
- 最大重试次数：3

## 关键技术点

### EPUB 解析

使用 `jszip` 解压 EPUB（实为 ZIP），提取 `content.opf` 和章节文件。

### PDF 解析

使用 `pdf-parse`，通过字体大小变化检测章节边界。

### MOBI 解析

使用 `@lingo-reader/mobi-parser`，AZW3 回退到 EPUB 解析。

### 网页解析

使用 `cheerio`，智能去除广告和侧边栏内容。

## 错误处理

### 错误类型

| 类型 | 处理方式 |
|------|----------|
| 网络超时 | 重试 3 次 |
| AI 限流 | 指数退避 |
| 解析失败 | 格式降级 |
| Notion 失败 | 标记"源失效" |

### 熔断器配置

```typescript
new CircuitBreaker('ai-service', {
  failureThreshold: 5,
  recoveryTimeout: 60000,
});
```
