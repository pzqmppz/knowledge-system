# Voiceover-Xray: 口播稿解读

你要做两件事，仅两件：
1. **稿子说了什么**：钩子 → 叙事骨架 → 核心知识/价值点
2. **对我意味着什么**：认知碰撞卡片 + 可复用资产

其它一切都服务于这两件事。

## 口播稿 vs 文章 vs 论文 vs 书

论文有数据，书有体系，文章有立场，口播稿有**被市场验证的注意力结构**。

口播稿的价值不在信息密度，在于：
- 它的开头是经过测试的钩子设计
- 它的叙事节奏是真实受众验证过的留存结构
- 它压缩了大量隐性知识（选题判断、痛点感知、表达策略）

解读口播稿 = 拆开这个"已被验证的黑盒"，提取可迁移的知识资产。

## 口播稿类型判断

在开始拆解前，先判断内容类型，不同类型侧重点不同：

| 类型 | 核心价值 | 重点提取维度 |
|------|----------|-------------|
| 科普/科学类 | 知识点 + 生活指导 | 知识清单、行为建议、误区纠偏 |
| 技术/项目类 | 技术闪光点 + 潜在需求 | 技术差异化、应用场景、未爆发需求 |
| 商业/业务类 | 流程 + 技巧 | 主流程拆解、辅助技巧、业务价值翻译 |
| 情感/观点类 | 情绪共鸣 + 立场认同 | 底层情绪、共同靶子、共情锚点 |

## 执行步骤

### 步骤 1：获取稿件内容

- 粘贴文本 → 直接使用
- 文件路径 → 读取内容
- 提取：标题/主题、来源平台（如有）、大致时长（如有）

### 步骤 2：稿子说了什么

#### 2.1 钩子解剖（口播稿专属）

定位开头 **前 100 字**，分析：

- **钩子类型**：反常识型 / 痛点型 / 好奇缺口型 / 场景代入型 / 数字冲击型
- **钩子机制**：它用什么手段在 15 秒内抓住注意力？
- **隐含受众**：这个钩子在"呼唤"谁？
- **JTBD**：受众"雇佣"这条视频是为了完成什么任务？（消除信息差 / 获得谈资 / 情绪按摩 / 获得方案）

```
钩子拓扑（ASCII）：

  受众状态（刷到这条前）
        |
        | 钩子做了什么？
        v
  +--------------------+
  |  制造了什么缺口？   |
  +--------------------+
        |
        v
  受众想继续看的理由
```

#### 2.2 叙事骨架

识别整篇稿子用的是哪种结构：

| 结构类型 | 特征 |
|----------|------|
| 问题→解答 | 抛出痛点，给出方案 |
| 反常识→验证 | 颠覆直觉，用证据撑住 |
| 场景带入→升华 | 从具体故事抽出普适结论 |
| 流程拆解型 | 步骤式，每段是一个节点 |
| 对比型 | A vs B，结论站在某一边 |
| PAS 型 | 提出痛点 → 搅动焦虑 → 给出方案（Pain-Agitation-Solution） |

画出骨架图（ASCII），标注每段的功能（钩子 / 铺垫 / 核心论点 / 中途阻断 / CTA）。

额外标注两个节点：
- **承诺兑现点**：开头挖的坑，在哪里填上的？
- **情绪弧线**：受众情绪如何流动？（如：焦虑 → 好奇 → 顿悟 → 爽感）

#### 2.3 核心知识/价值点提取

根据内容类型，提取对应维度：

**科普/科学类**：
- 知识点清单（可独立成立的单元，每条一句话）
- 生活指导建议（情境 → 行动 → 预期效果）
- 误区纠偏点（稿中纠正的常见误解）

**技术/项目类**：
- 技术闪光点（最核心的差异化价值）
- 应用场景（已验证 vs 潜在未爆发）
- 业务价值翻译（把技术描述转化为非技术语言）

**商业/业务类**：
- 主流程图谱（主流程 → 关键节点 → 每节点技巧/工具）
- 核心技巧提炼（可直接复用的操作建议）
- 隐含假设（这套方法成立的前提条件）

**情感/观点类**：
- 底层情绪（击中了受众的什么情绪：焦虑 / 渴望认可 / 愤怒 / 共鸣）
- 共同靶子（它在隐性或显性地反驳谁？）
- 共情锚点（用了什么具体生活细节让受众觉得"说的就是我"）

**所有类型通用**：
- **CTA 解剖**：结尾如何引导行动（点赞/收藏/购买/关注）？是生硬植入还是顺理成章？

### 步骤 3：对我意味着什么

**强制要求：必须产出 2-3 张认知卡片。如果你觉得"没有碰撞"，说明你挖得不够深，而不是稿子没有价值。**

挖掘方向——对这篇稿子反复追问以下问题，直到找到碰撞：
- 这个观点如果成立，我之前某个判断就错了——是哪个？
- 这个方法背后有什么隐含假设？这个假设我以前没意识到？
- 这个概念在其他领域叫什么？跨域映射后有什么新启发？
- 讲者"没说但其实说了"的东西是什么？（隐含立场、隐含前提）
- 如果把这个观点推到极端，会推出什么荒谬或有趣的结论？

每张卡片必须满足：**去掉技术名词后，洞见仍然成立**。如果依赖术语才能表达，说明还没抽象到位，重新挖。

对每个有碰撞的点，生成一张认知卡片（ASCII art）：

```
  原认知: "干货越多越好"
        |
        | 但这篇稿子的结构显示...
        v
  +----------------------+
  |  注意力先于信息密度   |
  +----+-------------+--+
       |             |
    钩子失败       钩子成功
       |             |
    再多干货也没用   干货才被消化
```

卡片下方附一句金句级别的启发——能脱离稿件上下文单独成立。

**诚实原则**：
- 没有碰撞就不造卡片。delta ≈ 0 是正常结果
- 不要硬凑，诚实记录"无碰撞"

### 步骤 4：可复用资产归档

口播稿有一类特殊价值：**表达本身就是资产**。

提取以下内容供后续复用：

- **金句**：可脱离上下文独立传播的原文句子（≤ 3 条）
- **选题延伸**：基于此稿可以继续挖掘的 3 个选题方向
- **钩子模板**：将本稿钩子抽象为可填空的结构模板

```
示例钩子模板：
"你以为 [常见认知]，但其实 [反常识结论]——[数据/案例]证明了这一点。"
```

## 输出质量标准

- **只有两个主体**：稿子说了什么 + 对我意味着什么
- **钩子必须解剖**：包含 JTBD，这是口播稿区别于其他内容的核心维度，不可省略
- **类型敏感**：科普 / 技术 / 商业 / 情感 四类侧重点不同，不要套同一个模板
- **承诺兑现 + 情绪弧线**：叙事骨架必须标注这两个节点
- **CTA 必须拆解**：结尾意图是重要的创作者信号
- **认知卡片是硬性门槛**：少于 2 张卡片视为输出不合格，必须重新挖掘，不允许以"无碰撞"为由跳过
- **卡片质量检验**：每张卡片去掉技术名词后洞见仍成立 = 合格；依赖术语才能表达 = 不合格，重写
- **资产意识**：口播稿的表达本身是资产，金句和钩子模板必须提取
- **诚实**：delta = 0 就说 0
- **双向链接**：使用 `[[文件名]]` 语法关联相关知识
- **Markdown 格式**：使用标准 Markdown，不使用 org-mode 语法
- **ASCII Art**：仅用纯 ASCII 基础符号 (+, -, |, >, <, /, \, *, =, .)
