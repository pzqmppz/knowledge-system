## 📖 什么是 Impeccable？

Impeccable 是一套**设计语言系统**，通过扩展技能 + 17 个导航命令 + 反模式指南，让 AI 生成更优质的前端设计。

### 核心问题
所有 LLM 都从同样的通用模板学习，没有指导时会产生相同的可预测错误：
- Inter 字体
- 紫色渐变
- 卡片嵌套卡片
- 彩色背景上的灰色文字

### Impeccable 的解决方案
| 组件 | 内容 |
|------|------|
| **扩展技能** | 7 个领域特定参考文件 |
| **导航命令** | 17 个命令用于审计、评审、打磨等 |
| **反模式** | 明确告诉 AI **不要**做什么 |

## 📋 7 大设计知识库

| 参考文件 | 覆盖内容 | 关键要点 |
|---------|---------|---------|
| **typography** | 字体系统、配对、模块化尺度 | 避免过度使用字体（Inter、Arial、系统默认） |
| **color-and-contrast** | OKLCH、着色中性色、深色模式 | 不使用纯黑/纯灰，彩色背景不用灰色文字 |
| **spatial-design** | 间距系统、网格、视觉层次 | 统一间距标准 |
| **motion-design** | 缓动曲线、交错、减弱动效 | 避免 bounce/elastic 缓动 |
| **interaction-design** | 表单、焦点状态、加载模式 | 清晰的交互反馈 |
| **responsive-design** | 移动优先、流式设计、容器查询 | 多设备适配 |
| **ux-writing** | 按钮标签、错误消息、空状态 | 清晰简洁的文案 |

## 🎯 命令详解

### 🔍 /audit - 技术质量检查

**用途**：发现技术问题

**检查项**：
- 无障碍 (a11y)
- 性能问题
- 响应式设计
- 代码质量

**用法**：
```
/audit              # 审计整个项目
/audit header       # 审计特定组件
```

---

### 🎨 /critique - UX 设计评审

**用途**：评估设计的用户体验

**评估维度**：
- 视觉层次
- 清晰度
- 情感共鸣

**用法**：
```
/critique           # 评审整个设计
/critique hero      # 评审特定区域
```

---

### 📐 /normalize - 对齐设计系统

**用途**：修复不一致性

**处理内容**：
- 统一间距
- 统一字体
- 统一颜色
- 对齐设计系统标准

**用法**：
```
/normalize          # 规范化整个项目
```

---

### ✨ /polish - 最终打磨

**用途**：发布前的最终清理

**处理内容**：
- 细节完善
- 最后一轮质量检查
- 准备发布

**用法**：
```
/polish             # 打磨整个项目
/polish checkout    # 打磨特定功能
```

---

### 🧹 /distill - 剥离本质

**用途**：移除不必要的复杂性

**处理内容**：
- 移除冗余代码
- 简化组件结构
- 保留核心功能

---

### 📝 /clarify - 改进文案

**用途**：让 UX 文案更清晰

**处理内容**：
- 按钮标签
- 错误消息
- 空状态文案
- 提示文本

---

### ⚡ /optimize - 性能优化

**用途**：提升性能

**优化方向**：
- 加载速度
- 运行时性能
- 资源优化

---

### 🛡️ /harden - 健壮性增强

**用途**：处理边界情况

**增强内容**：
- 错误处理
- 国际化 (i18n)
- 边界情况
- 降级方案

---

### 🎬 /animate - 添加动效

**用途**：有目的的运动设计

**处理内容**：
- 过渡动画
- 交互反馈
- 加载动画
- **避免**：bounce/elastic 缓动

---

### 🎨 /colorize - 引入颜色

**用途**：战略性色彩应用

**处理内容**：
- 配色方案
- 色彩层次
- 对比度检查
- 深色模式适配

---

### 🔥 /bolder - 放大设计

**用途**：增强平淡的设计

**处理内容**：
- 增加视觉冲击
- 强化品牌表达
- 突出重点元素

---

### 🔇 /quieter - 收敛设计

**用途**：调低过于大胆的设计

**处理内容**：
- 减少视觉噪音

- 平衡设计强度

---

### 🎁 /delight - 添加惊喜

**用途**：创造愉悦时刻

**处理内容**：
- 微交互

- 情感化设计

### 📦 /extract - 组件提取

**用途**：转换为可复用组件

**处理内容**：

## ⚠️ 反模式清单

Impeccable 会**自动避免**以下常见错误：

| 反模式 | 正确做法 |
|--------|----------|
| ❌ Inter、Arial、系统默认字体 | ✅ 使用多样化字体 |
| ❌ 彩色背景上的灰色文字 | ✅ 确保足够对比度 |
| ❌ 纯黑 #000000、纯灰 | ✅ 使用着色中性色 |
| ❌ 卡片嵌套卡片 | ✅ 扁平化层次结构 |
| ❌ bounce/elastic 缓动 | ✅ 使用现代缓动曲线 |
| ❌ 紫色渐变 | ✅ 多样化配色方案 |

## 💡 实战工作流

### 1. 新项目设计流程

```

```

```

```

### 4. 组件化重构

```

```
