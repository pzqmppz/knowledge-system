## 四大开发模式

```
┌────────────────────────────────────────────────────────────────────────┐
│                         VIBE CODING v2.0                              │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  📚 学习模式 (LEARN)    →  手动微循环，适合新手学习流程                 │
│  ⚡ 加速模式 (FAST)      →  质量门控增强，适合熟练开发者                │
│  🚀 自动模式 (AUTO)      →  完全自主循环，适合专家释放时间              │
│  🎯 EDD 模式 (EVAL)      →  评估驱动开发，适合需要严格质量保证的项目    │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### 模式对比

| 模式 | 自动化程度 | 对话轮数 | 适用场景 | 核心 Commands | 详细文档 |
|------|-----------|---------|----------|---------------|----------|
| 📚 学习模式 | 手动 | 40-50 轮/功能 | 新手学习、理解原理 | /plan, /tdd-workflow | [[02a-学习模式]] |
| ⚡ 加速模式 | 半自动 | 15-20 轮/功能 | 熟练开发、提高效率 | /quality-gate, /plankton-code-quality | [[02b-加速模式]] |
| 🚀 自动模式 | 全自动 | 2-3 轮/功能 | 专家使用、重复性任务 | /loop-start, /autonomous-loops | [[02c-自动模式]] |
| 🎯 EDD 模式 | 评估驱动 | 可变 | 需要严格质量保证 | /eval-harness | [[02d-EDD模式]] |

## 🎯 模式选择决策树

```
是否第一次使用 VIBE CODING？
├─ 是 → 📚 [[02a-学习模式]]
│   └─ 理解流程、掌握技巧
└─ 否 → 需要严格质量保证？
    ├─ 是 → 🎯 [[02d-EDD模式]]
    │   └─ 评估驱动、数据化决策
    └─ 否 → 功能是否重复性高？
        ├─ 是 → 🚀 [[02c-自动模式]]
        │   └─ 完全自主、节省时间
        └─ 否 → ⚡ [[02b-加速模式]]
            └─ 半自动、质量保证
```
