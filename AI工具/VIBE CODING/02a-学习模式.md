## 📚 模式 A：学习模式（手动微循环）

**适用：** 新手学习、理解流程原理

**特点：** 完全手动控制，每步确认

**预计对话：** 40-50 轮/功能

---

## 微循环 6 步详解

```
┌─────────────────────────────────────────────┐
│  一次微循环（学习模式）                       │
├─────────────────────────────────────────────┤
│ 对话 1: "/tdd [功能描述]"                    │
│         ↓                                   │
│         (自动完成 Red + Green)               │
├─────────────────────────────────────────────┤
│ 对话 2: "/code-review"                      │
│         ↓                                   │
│         → 如果有 CRITICAL：立即修复          │
│         → 如果是建议：记录，继续             │
├─────────────────────────────────────────────┤
│ 对话 3: "/verify quick"                     │
│         ↓                                   │
│         → 如果失败：/build-fix               │
│         → 如果通过：人工测试                 │
├─────────────────────────────────────────────┤
│ 对话 4: (人工验收)                          │
│         "测试通过" 或 "有问题，修复XX"       │
├─────────────────────────────────────────────┤
│ 对话 5: "/checkpoint create 'feat: ...'"    │
│         ↓                                   │
│         (锁定胜局)                           │
└─────────────────────────────────────────────┘

总计：4-5 轮对话，然后进入下一个循环
```

##### Phase 1: 基础架构 - 最小实现

##### 实现 API 客户端 (lib/api.ts)
```typescript
import { SearchQuery, SearchResult } from '@/types'

export async function searchAPI(
  query: SearchQuery
): Promise<SearchResult[]> {
  // 处理空查询
  if (!query.keyword.trim()) {
    return []
  }

  try {
    const response = await fetch(`/api/search?q=${query.keyword}`)

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`)
    }

    const data = await response.json()
    return data.results || []
  } catch (error) {
    console.error('Search failed:', error)
    return []
  }
}
```

## 学习模式的优势

1. **深度理解**：每步都理解为什么这么做
2. **建立习惯**：通过重复建立最佳实践习惯
3. **问题定位**：出现问题能快速定位到具体步骤
4. **知识沉淀**：更容易形成可复用的开发模式

## 学习模式的局限

1. **时间成本高**：40-50 轮对话/功能
2. **效率较低**：不适合紧急项目
3. **需要耐心**：每步都需要确认

---

> **💡 使用建议：**
>
> **什么时候使用学习模式**：
> ✅ 第一次使用 VIBE CODING
> ✅ 学习新的技术栈
> ✅ 需要深度理解项目架构
> ✅ 团队新成员培训
>
> **什么时候切换到其他模式**：
> ⚠️ 已熟悉流程（切换到加速模式）
> ⚠️ 时间紧张（切换到自动模式）
> ⚠️ 重复性任务（切换到自动模式）
