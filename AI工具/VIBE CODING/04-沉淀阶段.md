[Hook] Session has 89 messages - evaluating for extractable patterns

##### 1. 防抖 Hook 实现 ⭐ 重要
- **问题**：useDebounce 在 strict mode 下触发两次
- **根因**：useEffect cleanup 未正确处理
- **解决方案**：
```typescript
useEffect(() => {
  const handler = setTimeout(() => {
    setDebouncedValue(value)
  }, delay)

  return () => clearTimeout(handler) // 关键！
}, [value, delay])
```
- **置信度**：High
- **复用性**：高

##### 2. TypeScript 泛型类型推断 ⭐ 重要
- **问题**：fetch 返回类型为 any
- **解决方案**：显式声明泛型
```typescript
const response = await fetch<SearchResult[]>(url)
```
- **置信度**：Medium
- **复用性**：高

##### 3. API 超时处理模式
- **问题**：网络请求可能长时间挂起
- **解决方案**：AbortController 超时控制
```typescript
const controller = new AbortController()
const timeout = setTimeout(() => controller.abort(), 5000)
```
- **置信度**：Medium
- **复用性**：高

```typescript
// ✅ 标准模式：useSearch hook
export function useSearch(endpoint: string) {
  const [query, setQuery] = useState('')
  const [results, setResults] = useState([])
  const [loading, setLoading] = useState(false)

  const debouncedQuery = useDebounce(query, 500)

  useEffect(() => {
    if (!debouncedQuery) {
      setResults([])
      return
    }

    setLoading(true)
    searchAPI({ keyword: debouncedQuery })
      .then(setResults)
      .finally(() => setLoading(false))
  }, [debouncedQuery])

  return { query, setQuery, results, loading }
}
```
