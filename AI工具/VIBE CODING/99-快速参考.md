## 立即可用的 Prompt 模板

### 【启动新项目】

```markdown
我要开发 [功能名称]，使用 vibe coding v2.0。

请：
1. 运行 /harness-audit 检查配置
2. 帮我选择合适的开发模式
3. 配置 /cost-aware-llm-pipeline（预算：$X）
4. 生成 PRD
5. 制定开发计划（每 Phase 50-80 行代码）
6. 等我确认后，按选定模式开始执行

不要一次性生成所有代码！
```

### 【切换开发模式】

```markdown
切换到 [模式名称] 继续开发。

当前进度：Phase [N] 已完成
下一步：Phase [N+1] - [描述]

请按 [模式名称] 的流程继续执行。
```

### 【启动自动模式】

```markdown
启动 🚀 自动模式完成剩余开发。

当前计划：[列出剩余 Phase]

要求：
- 自动执行所有 Phase
- 质量门控实时监控
- 遇到 CRITICAL 时暂停
- 每个 Phase 自动 checkpoint

开始执行。
```

### 【触发调试模式】

```markdown
遇到问题：[描述问题]

请：
1. 启动 /nanoclaw-repl 调试
2. 定位问题根因
3. 提供修复方案
4. 验证修复效果
```

### 【完全自主开发】

```markdown
我要完全自主开发 [功能描述]。

请：
1. /harness-audit
2. 选择最合适的模式
3. 配置成本优化（预算：$X）
4. 生成 PRD 并等我确认
5. 确认后启动 /loop-start 完全自主执行
6. 完成后运行 /verification-loop
7. 生成最终报告

目标：最小人工干预，最高质量输出。
```

## 阶段快速导航

### 🟢 启动阶段
**文档**：[[01-启动阶段]]
**目标**：选择开发模式，完成项目初始化
**时间**：30-50 分钟
**关键步骤**：
1. 模式选择
2. Harness 审核
3. 技能预加载
4. 成本优化配置
5. 需求共创
6. 工程初始化
7. 架构重构
8. 制定计划

### 🟡 执行阶段
**文档**：[[02-执行阶段]]
**目标**：根据选择的模式执行开发任务
**时间**：1-3 小时
**四种模式**：
- [[02a-学习模式]]：手动微循环（40-50 轮对话）
- [[02b-加速模式]]：质量门控增强（15-20 轮对话）
- [[02c-自动模式]]：完全自主循环（2-3 轮对话）
- [[02d-EDD模式]]：评估驱动开发（可变）

### 🔴 验证阶段
**文档**：[[03-验证阶段]]
**目标**：确保代码符合生产标准
**时间**：30-45 分钟
**关键步骤**：
1. 质量门控
2. 综合验证
3. 多 Agent 审查
4. 最终验收 (UAT)

### 🟣 沉淀阶段
**文档**：[[04-沉淀阶段]]
**目标**：沉淀经验，持续改进
**时间**：10-15 分钟
**关键步骤**：
1. 自动经验提取
2. 项目 Skill 更新
3. 文档同步

### 🔵 调试阶段
**文档**：[[05-调试阶段]]
**目标**：快速定位和修复问题

## 核心原则总结

### v2.0 新增原则

1. ✅ **模式选择优先** - 根据场景选择最合适的模式
2. ✅ **质量门控驱动** - 自动化质量检查，而非人工审查
3. ✅ **成本意识** - 优化 API 调用，控制预算
4. ✅ **自动化优先** - 能自动的绝不手动
5. ✅ **评估驱动** - 数据化决策，而非主观判断
6. ✅ **持续学习** - 自动提取经验，而非手动记录

### v1.0 保留原则

1. ✅ **测试驱动** - 先写测试，再写代码
2. ✅ **小步迭代** - 每 50 行一个循环
3. ✅ **持续验证** - 每步都要 verify
4. ✅ **经验沉淀** - 用 /learn 保存模式

### C. 常见问题

#### Q1: 如何选择开发模式？
参考 [[02-执行阶段]] 的模式选择指南，或使用决策树。

#### Q2: 成本如何控制？
使用 `/cost-aware-llm-pipeline` 配置预算，选择合适的模式，使用自动模式可以节省 30-40% 成本。

#### Q3: 如何保证代码质量？
使用质量门控（`/quality-gate`）、写时代码质量检查（`/plankton-code-quality`）、综合验证（`/verification-loop`）。

#### Q4: 出现问题怎么办？
参考 [[05-调试阶段]]，使用 `/nanoclaw-repl` 快速定位和修复问题。

#### Q5: 如何提升开发效率？
- 熟悉流程后使用 ⚡ 加速模式
- 重复性任务使用 🚀 自动模式
- 完善的项目 Skill 可以减少重复解释
