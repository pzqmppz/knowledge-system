# 知识体系维护计划

> 本文档定义知识体系的定期维护任务和检查清单

---

## 维护周期

### 每次操作后

- [ ] 检查文件元数据是否完整
- [ ] 验证双向链接是否有效
- [ ] 确认索引已更新

### 每周 (周日)

- [ ] 检查失效链接
- [ ] 更新过时内容
- [ ] 审查 orphan 文件
- [ ] 清理 box/ 目录

### 每月 (24日)

- [ ] 审查分类结构
- [ ] 同步 config.yaml
- [ ] 更新 README.md 统计
- [ ] 备份知识体系
- [ ] 审核操作准则

### 每季度

- [ ] 大规模整理
- [ ] 合并重复内容
- [ ] 优化标签体系
- [ ] 更新文档版本

---

## 检查清单

### 配置同步检查

```
config.yaml vs 实际目录
├── categories 列表与实际目录一致
├── 新增分类已添加描述
└── 忽略目录配置正确

README.md vs 实际结构
├── 目录结构说明准确
├── 配置文件链接有效
├── Obsidian 说明最新
└── 统计数据正确

INDEX.md 统计
├── 各分类文件数正确
├── 最近更新日期准确
└── 标签云完整

TAGS.md 标签
├── 按字母顺序排列
├── 文件计数准确
└── 新标签已添加
```

### 内容质量检查

```
文件完整性
├── YAML Frontmatter 完整
├── 必填字段齐全
├── 双向链接有效
└── 反向链接区域存在

内容时效性
├── 标记过时内容
├── 更新失效链接
└── 补充缺失信息
```
