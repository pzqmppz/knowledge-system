# 书籍处理规范

> 定义书籍从下载到知识点归档的完整流程

---

## 核心原则

**拆解优先，按需精读，内化核心**

1. 通过 Skill 拆解书籍，提取核心概念
2. 优先阅读拆解后的内容
3. 根据需要选择性精读原书章节
4. 将核心知识点归档到知识体系

---

## 书籍生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                     书籍处理流程                             │
└─────────────────────────────────────────────────────────────┘

1️⃣ 下载书籍
   box/书库/[书名].[epub/pdf/mobi]
        ↓
2️⃣ 使用 Skill 拆解
   生成拆解文件（由 Skill 处理）
        ↓
3️⃣ 优先阅读拆解内容
   快速获取核心概念、摘要、要点
        ↓
4️⃣ 按需精读原书
   根据拆解指引，只读需要的章节
        ↓
5️⃣ 核心知识点归档
   提取到对应分类目录
        ↓
6️⃣ 书籍处理完成
   可选择保留或删除原书文件
```

## 存储结构

### 临时存储（box/）

```
box/书库/
├── [书名].[epub/pdf/mobi]     # 原始书籍文件
└── ...
```

### 知识点归档（各分类/）

```
(各分类)/
├── [书名]-核心概念.md           # 提取的核心知识点
├── [书名]-阅读笔记.md           # 阅读时的笔记和思考
└── [书名]-拆解报告.md           # 完整拆解报告（可选）
```

## 拆解状态定义

| 状态 | 说明 |
|------|------|
| `待拆解` | 书籍已下载，等待使用 Skill 拆解 |
| `已拆解` | Skill 已处理，可阅读拆解内容 |
| `阅读中` | 正在阅读拆解内容或原书 |
| `已完成` | 核心知识点已提取归档 |

---

## 快速操作指南

### 添加新书籍

1. 下载书籍到 `box/书库/`
2. 在 `box/INDEX.md` 中登记
3. 使用 Skill 拆解书籍
4. 创建阅读笔记文件
5. 核心知识点归档到对应分类

### 查找书籍

1. 查看 `box/INDEX.md` 索引
2. 或在知识体系各分类中查找已提取的知识点
