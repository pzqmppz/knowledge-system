# GitHub 项目分析规范

> 定义 GitHub 项目从发现到知识点归档的完整流程

---

## 核心原则

**分析提取，主题分类，知识归档**

1. 发现值得学习的 GitHub 项目
2. 记录到 `编程项目/项目讨论/`
3. 深入分析项目架构和代码
4. 按主题提取知识点归档到对应分类
5. 完成后移动到 `编程项目/已完成/`

---

## 项目生命周期

```
┌─────────────────────────────────────────────────────────────┐
│                  GitHub 项目处理流程                         │
└─────────────────────────────────────────────────────────────┘

1️⃣ 发现项目
   项目链接、Star 数、技术栈
        ↓
2️⃣ 创建分析记录
   编程项目/项目讨论/[项目名]-分析/
        ↓
3️⃣ 深入分析
   架构设计、源码阅读、技术亮点
        ↓
4️⃣ 知识点提取
   按主题分类归档到各目录
        ↓
5️⃣ 项目总结
   完成后移动到 编程项目/已完成/
```

## 存储结构

### 临时存储（box/项目/）

```
box/项目/
└── [项目名]/
    ├── README.md              # 项目说明
    ├── source/                # 源代码（不上传 Git）
    └── docs/                  # 项目文档
```

### 分析过程（编程项目/项目讨论/）

```
编程项目/项目讨论/[项目名]-分析/
├── 项目概况.md
├── 架构分析.md
├── 核心模块.md
├── 技术亮点.md
└── 知识点/
    ├── 提取的知识点1.md → 归档到相关分类
    └── 提取的知识点2.md → 归档到相关分类
```

### 已完成归档（编程项目/已完成/）

```
编程项目/已完成/
└── [项目名]/
    ├── 项目说明.md
    ├── 项目复盘.md
    └── 技术文档.md
```

## 项目分析元数据

```yaml
---
title: "[项目名] 源码分析"
category: "相关分类"
tags: ["github", "项目分析", "技术栈"]
project:
  repo: "owner/repo"
  url: "https://github.com/owner/repo"
  stars: 数量
  language: "主要语言"
  license: "许可证"
  status: "analyzing"        # analyzing / completed
focus:
  - "关注点1"
  - "关注点2"
analysis:
  architecture: [[相关分类/架构笔记]]
  core_modules: [[相关分类/模块笔记]]
  techniques: [[相关分类/技术笔记]]
---
```

## 知识点提取规则

### 按主题分类

| 项目类型 | 提取方向 | 归档分类 |
|----------|----------|----------|
| **前端框架** | 架构设计、核心模块、Hooks实现 | 前端开发 |
| **后端框架** | 架构设计、路由、中间件 | 后端开发 |
| **工具库** | 使用方法、最佳实践 | 工具与效率 |
| **算法库** | 算法设计、数据结构 | 算法与数据结构 |
| **数据库** | 数据库设计、查询优化 | 数据库 |

## 快速操作指南

### 添加新项目

1. 发现项目，记录到 `编程项目/项目讨论/`
2. 创建分析文件夹
3. 使用 Skill 或手动分析项目
4. 提取知识点并归档
5. 完成后移动到 `已完成/`

### 查找项目分析

1. 查看 `编程项目/INDEX.md`
2. 或在各分类中查找已提取的知识点
