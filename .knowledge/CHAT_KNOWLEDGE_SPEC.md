# 日常聊天知识点梳理规范

> 本文档定义从日常对话中提取知识点并生成知识卡片的规范

---

## 适用场景

- 与 AI 助手的对话中产生的有价值知识点
- 问题解决过程中形成的实践经验
- 技术讨论中总结的概念和方法
- 临时学习笔记的整理和归档

---

## 知识卡片生成流程

### 1. 识别知识点

从对话中识别以下类型的知识点：

| 类型 | 说明 | 示例 |
|------|------|------|
| **概念解释** | 技术概念的定义和解释 | CUDA 是什么 |
| **操作指南** | 步骤化的操作流程 | Docker 数据迁移步骤 |
| **对比分析** | 多个选项的对比 | 多模型部署复用层级 |
| **问题解决** | 常见问题和解决方案 | 镜像空壳问题 |
| **最佳实践** | 推荐的做法和模式 | 目录结构设计 |

### 2. 确定分类

根据知识点的主题确定归属分类：

```
AI工具/          - AI 相关工具和技术
DevOps运维/      - 运维、容器、部署
后端开发/        - 后端技术
前端开发/        - 前端技术
数据库/          - 数据库相关
计算机基础/      - 基础概念
工具与效率/      - 效率工具
```

### 3. 生成知识卡片

按照以下模板生成知识卡片：

---

## 双向链接规范

### 何时添加双向链接

| 情况 | 示例 |
|------|------|
| 提到已有知识点 | [[Docker]]、[[CUDA]] |
| 引用其他文件 | [[../DevOps运维/Docker入门]] |
| 关联相关概念 | [[vLLM]]、[[模型部署]] |

### 链接语法

```markdown
# 基本链接
[[文件名]]

# 带别名的链接
[[文件名|显示名称]]

# 链接到其他分类
[[../分类名/文件名]]

# 链接到标题
[[文件名#章节标题]]
```

---

## 反向链接区域

每个知识卡片末尾应包含反向链接区域：

```markdown
---

## 🔗 反向链接

### 前置知识
- [[概念A]] - 理解本文需要先掌握的知识

### 相关内容
- [[文件B]] - 主题相关可对比学习

### 应用场景
- 具体的应用场景描述
```

### 反向链接类型

| 类型 | 用途 | 语法 |
|------|------|------|
| **前置知识** | 理解当前内容需要先掌握的知识 | `[[概念]]` |
| **相关内容** | 主题相关、可以对比学习的内容 | `[[文件]]` |
| **应用场景** | 理论知识在实际中的应用 | 描述文本 |
| **扩展阅读** | 更深入或更高级的内容 | `[[文件]]` |

## 标签规范

### 必选标签

- 分类标签：`#ai`、`#docker`、`#python` 等
- 来源标签：`#chat`（日常对话）

### 可选标签

| 类型 | 示例 |
|------|------|
| 技术标签 | `#cuda`, `#vllm`, `#gpu` |
| 难度标签 | `#beginner`, `#intermediate`, `#advanced` |
| 状态标签 | `#learned`, `#practice` |
| 来源标签 | `#article`, `#book`, `#chat`, `#practice` |

### 层级标签

使用 `/` 表示层级：

```
#ai/deploy        → AI 部署相关
#docker/volume    → Docker 卷相关
#gpu/cuda         → GPU CUDA 相关
```
