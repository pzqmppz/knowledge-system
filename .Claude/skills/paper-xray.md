# Paper-Xray: 论文解读

你要做两件事，仅两件：
1. **论文说了什么**：问题 → 视角 → 结果
2. **对我意味着什么**：认知卡片（ASCII art 直观展示启发）

其它一切都服务于这两件事。

## 知识体系结构

```
知识体系/
├── box/                    # 临时存储（git-ignored）
│   └── 论文/              # 原始论文存储
│       └── {论文文件}
├── {分类目录}/             # 知识归档
│   └── {论文标题}-解读.md   # 解读结果
└── box/INDEX.md           # 索引更新
```

## 执行步骤

### 步骤 1：接收论文并预处理输入

| 用户输入 | 转换规则 |
|---------|---------|
| `2601.01290` 或 `arxiv:2601.01290` | → `https://arxiv.org/html/2601.01290` |
| `https://arxiv.org/abs/...` 或 `.../pdf/...` | → 替换为 `/html/` |
| `https://arxiv.org/html/...` | 直接使用 |
| 其他 URL 或 PDF 路径 | 按原有逻辑处理 |

arxiv 论文一律转为 `/html/` 格式，HTML 版本可直接 WebFetch 抓取全文。

### 步骤 2：论文说了什么

像跟聪明朋友在饭桌上说"这篇论文干了个什么事"——三句话，人话，不要论文八股：
- **问题**：作者要解决什么？
- **视角**：用什么方法/切入角度？
- **结果**：得到了什么？用"实验证明"和"作者推测"自然区分。推测性结论流入"追问"区

**零术语规则**：每个技术概念必须立刻落在读者见过的事情上。不给例子就不准用那个词。

坏："共现概率只取决于隐含空间中的距离（平移对称性）"
好："一月和二月经常出现在同一段话里，一月和七月就很少——距离越近越常一起出现"

**承重概念场景化**：每篇论文有 2-3 个"承重概念"——去掉它们论文的论证就塌了。对每个承重概念：

1. 先找一个读者亲身经历过的场景
2. 在场景中逐步展开——至少 3 级渐进
3. 最后才贴技术名字

判断标准：去掉技术名字，读者仍然知道你在说什么 = 落地成功。

配一张餐巾纸图（ASCII），画出核心机制。

### 步骤 3：对我意味着什么

读完这篇论文，我带走什么？

任何能提升我的决策质量的洞见都值得一张卡片：
- 一个可以直接用的思维工具（之前没有的）
- 一个改变了某个判断的新证据
- 一个没想过的角度或盲区
- 一个对已有认知的补充或修正

对每个有启发的点，生成一张认知卡片：

```
认知卡片 = 一张 ASCII art
```

卡片要直观展示：这个洞见如何改变我的某个思考方式、决策习惯、或打开一个盲区。

卡片下方附一句金句级别的启发——能脱离论文上下文单独成立。

**好卡片标准**：空间关系本身在说话，看排列就能懂。

```
  原认知: "做了才懂"
        |
        |  但如果结构藏得太深...
        v
  +------------------+
  | 反馈能暴露结构?   |
  +---+----------+---+
      |          |
     YES         NO
      |          |
   生成优先    降秩优先
      |          |
      v          v
   原认知成立   需加边界
```

**诚实原则**：
- 没有碰撞就不造卡片。delta ≈ 0 是正常结果
- 论文的推测性观点如果有趣，放进"追问"区

### 步骤 4：确定归档分类

根据论文主题，选择合适的分类目录：

| 论文类型 | 归档目录 |
|----------|----------|
| AI/ML 论文 | AI工具/ 或 CS基础/ |
| 算法论文 | 算法与数据结构/ |
| 系统论文 | 系统设计/ 或 后端开发/ |
| 数据库论文 | 数据库/ |
| 前端论文 | 前端开发/ |
| 其他 | 其他/ |

### 步骤 5：生成 Markdown 报告

使用 Write 工具，生成 Markdown 文件。

**文件路径**：`{分类目录}/{论文标题}-解读.md`

**模板**：

```markdown
---
title: "{论文标题} - 解读"
category: "{分类}"
tags: ["论文", "解读", "{主题标签}"]
type: "论文解读"

paper_authors: "{作者列表}"
paper_venue: "{会议/期刊}"
paper_year: {年份}

paper_arxiv_id: "{arxiv ID（如有）}"
pdf_source: "box/论文/{原始文件名}"
```

## 输出质量标准

- **只有两个部分**：论文说了什么 + 对我意味着什么
- **承重概念场景化**：用3级渐进展开，不是括号注释
- **认知卡片 = ASCII art**：视觉优先，不是文字列表
- **诚实**：delta = 0 就说 0
- **双向链接**：使用 `[[文件名]]` 语法关联相关知识
- **Markdown 格式**：使用标准 Markdown
