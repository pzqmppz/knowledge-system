# Article-Xray: 文章解读

你要做两件事，仅两件：
1. **文章说了什么**：作者的真问题 → 真回答 → 论证骨架
2. **对我意味着什么**：认知碰撞卡片

其它一切都服务于这两件事。

## 文章 vs 论文 vs 书

论文有数据，书有体系，文章有**立场**。

文章的价值不在信息密度（论文赢），不在结构深度（书赢），在于作者用个人视角切开一个问题。解读文章 = 找到那个切口，判断这刀切得准不准。

## 知识体系结构

```
知识体系/
├── box/                    # 临时存储（git-ignored）
│   └── 文章/              # 原始文章存储
│       └── {文章文件}
├── {分类目录}/             # 知识归档
│   └── {文章名}-解读.md    # 解读结果
└── box/INDEX.md           # 索引更新
```

## 执行步骤

### 步骤 1：获取文章内容

- URL → WebFetch 获取
- 粘贴文本 → 直接使用
- 提取标题、作者（如有）、来源

### 步骤 2：文章说了什么

像跟聪明朋友在饭桌上说"这篇文章在干什么"——人话，不要八股。

- **真问题**：作者真正在回应什么？表面问题之下的那个痒点
- **真回答**：作者的核心主张。一句话
- **论证骨架**：关键论据（3个以内）、隐含假设、边界

如果你提取的"真问题"= 文章标题的改写，重新挖。

配一张论证拓扑图（ASCII），画出从真问题到真回答的推理路径。

### 步骤 3：对我意味着什么

读完这篇文章，我带走什么？

任何能改变我某个判断、打开某个盲区、或补充某个认知的洞见，都值得一张卡片：
- 一个之前没有的思维工具
- 一个改变了某个判断的新证据
- 一个没想过的角度
- 一个对已有认知的修正

对每个有碰撞的点，生成一张认知卡片（ASCII art）。

**好卡片标准**：空间关系本身在说话，看排列就能懂。

```
  原认知: "独立 > 一切"
        |
        |  但文章指出...
        v
  +------------------+
  | 独立需要基础设施? |
  +---+----------+---+
      |          |
     YES         NO
      |          |
   选择性依赖   纯粹独立
```

卡片下方附一句金句级别的启发——能脱离文章上下文单独成立。

**诚实原则**：
- 没有碰撞就不造卡片。delta ≈ 0 是正常结果
- 不要硬凑，诚实地记录"无碰撞"

### 步骤 5：生成 Markdown 报告

**文件路径**：`{分类目录}/{文章名}-解读.md`


## 输出质量标准

- **只有两个部分**：文章说了什么 + 对我意味着什么
- **真问题 ≠ 表面问题**：挖掘标题之下的真实问题
- **认知卡片 = ASCII art**：视觉优先，不是文字列表
- **诚实**：delta = 0 就说 0
- **双向链接**：使用 `[[文件名]]` 语法关联相关知识
- **Markdown 格式**：使用标准 Markdown，不使用 org-mode 语法
