# 知识链接同步系统

> 用于解决文件移动/重命名后双向链接失效的问题

## 问题说明

当文件移动或重命名时，原有的 `[[文件名]]` 链接会失效。

**示例**：
- 原路径：`拆解报告/书籍/XXX-拆解报告.md`
- 新路径：`书籍管理/阅读中/书库/XXX-拆解报告.md`
- 失效链接：`[[书籍/XXX-拆解报告]]`

---

## 解决方案

### 方案1: Obsidian 自动更新（推荐日常使用）

1. 打开 Obsidian 设置
2. 进入 **文件与链接** → **链接**
3. 启用以下选项：
   - ✅ **自动更新内部链接**
   - ✅ **检测重命名时更新链接**
   - ✅ **使用路径别名**

**效果**：当你在 Obsidian 中重命名/移动文件时，所有相关链接自动更新。

---

### 方案2: 链接同步脚本（批量修复）

运行以下命令批量修复失效链接：

```bash
# 方式1: 使用 sed (Linux/Mac/WSL)
find . -name "*.md" -type f -exec sed -i 's/旧链接/新链接/g' {} +

# 方式2: 使用 Python (跨平台)
python scripts/fix-links.py
```

**脚本位置**：`.Claude/scripts/fix-links.py`

---

### 方案3: 唯一 ID 系统（长期方案）

为每个笔记分配不依赖路径的唯一 ID。

**ID 格式**：
```yaml
---
id: "20260303-001"  # YYYYMMDD-序号
---
```

**链接方式**：
```markdown
[[id:20260303-001|显示名称]]
```

**优势**：
- 文件可以随意移动/重命名
- 链接永不失效
- 适合长期知识管理

---

## 使用建议

| 场景 | 推荐方案 |
|------|----------|
| 日常编辑 | 方案1 (Obsidian 自动更新) |
| 大规模重构 | 方案2 (批量脚本) |
| 新建知识库 | 方案3 (唯一 ID) |
